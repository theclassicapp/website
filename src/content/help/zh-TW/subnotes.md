# 子筆記

子筆記讓您可以在 Classic 中建立巢狀、階層式的筆記。將複雜主題組織成易於管理的部分。

## 什麼是子筆記？

子筆記是筆記中的筆記——一種建立階層式內容結構的方式。

### 優點

- **組織**：將複雜主題拆分為較小的部分
- **巢狀**：建立多層級的階層結構
- **導航**：輕鬆在相關筆記之間導航
- **彈性**：依需求展開和折疊

## 建立子筆記

### 方法 1：拖放

透過將內容拖放到父筆記下建立子筆記：

1. 建立一個父筆記
2. 建立一個新筆記
3. 將新筆記拖曳到父筆記上
4. 它會變成子筆記

### 方法 2：右鍵選單

1. 右鍵點擊筆記
2. 選擇「新增子筆記」
3. 巢狀筆記即建立

### 方法 3：快速鍵

1. 選擇一個筆記
2. 按 `Cmd/Ctrl+Shift+N`
3. 子筆記即建立

## 管理子筆記

### 展開與折疊

點擊帶有子筆記的筆記旁邊的箭頭：

- **展開**：顯示所有巢狀筆記
- **折疊**：隱藏巢狀筆記

### 導航

- 點擊以開啟子筆記
- 使用方向鍵導航階層
- 麵包屑導航顯示您的位置

### 移動子筆記

拖放以重新組織：

1. 點擊並按住子筆記
2. 拖曳到新位置
3. 放下以重新定位

### 轉換為主筆記

將子筆記提升為頂層：

1. 右鍵點擊子筆記
2. 選擇「移至根目錄」
3. 筆記變成主筆記

## 子筆記深度

Classic 支援多層級的巢狀結構：

```
📁 專案
  📄 概述
  📁 研究
    📄 使用者訪談
    📄 問卷結果
    📁 分析
      📄 關鍵發現
      📄 建議
  📁 規劃
    📄 時程
    📄 資源
```

### 最佳實踐

- **限制深度**：最多 3-4 層以保持清晰
- **邏輯分組**：將相關內容歸類在一起
- **清楚命名**：為子筆記使用描述性名稱

## 與子筆記連結

### 反向連結

子筆記會自動建立到父筆記的反向連結：

- 查看「連結參考」區塊
- 在相關筆記之間導航
- 了解筆記關係

### Wiki 連結

從任何地方連結到子筆記：

```markdown
詳見 [[專案/研究/分析]]。
```

## 提示

### 命名慣例

使用清楚、一致的名稱：

```
✅ 良好：
專案/需求/功能面
專案/需求/非功能面

❌ 避免：
專案/req/func
專案/rq/nf
```

### 麵包屑導航

使用麵包屑了解您的位置：

```
專案 > 研究 > 分析
```

### 搜尋

搜尋會自動包含子筆記：

- 在所有層級中尋找內容
- 依筆記深度篩選
- 快速導航

### 範本

將範本套用到子筆記：

1. 建立子筆記
2. 使用斜線命令 `/template`
3. 選擇範本

## 快速鍵

| 操作 | 快速鍵 |
|------|--------|
| 新增子筆記 | `Cmd/Ctrl+Shift+N` |
| 全部展開 | `Cmd/Ctrl+Down` |
| 全部折疊 | `Cmd/Ctrl+Up` |
| 上移 | `Opt/Alt+Up` |
| 下移 | `Opt/Alt+Down` |
| 提升 | `Opt/Alt+Left` |
| 降級 | `Opt/Alt+Right` |

## 疑難排解

### 子筆記未顯示

- 檢查父筆記是否已折疊
- 確認筆記未被移動
- 重新整理側邊欄

### 無法建立子筆記

- 確保父筆記存在
- 檢查權限
- 嘗試右鍵選單方法

### 子筆記遺失

使用搜尋尋找遺失的筆記：

1. 開啟搜尋（`Cmd/Ctrl+F`）
2. 輸入筆記名稱
3. 在結果中查看完整路徑
