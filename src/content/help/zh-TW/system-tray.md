# 系統匣

Classic 位於系統匣中以便快速存取。即時捕捉想法而不中斷工作流程。

## 什麼是系統匣？

系統匣（Windows/Linux）或選單列（macOS）是 Classic 在背景執行的地方，隨時準備快速擷取。

### 優點

- **隨時可用**：一鍵即達
- **快速擷取**：即時記下想法
- **背景執行**：無需保持應用程式開啟
- **通知**：獲得任務提醒

## 啟用系統匣

### macOS

1. 開啟 Classic
2. 前往設定 → 一般
3. 啟用「在選單列中顯示」
4. Classic 圖示會出現在選單列中

### Windows

1. 開啟 Classic
2. 前往設定 → 一般
3. 啟用「在系統匣中顯示」
4. Classic 圖示會出現在系統匣中

### Linux

1. 開啟 Classic
2. 前往設定 → 一般
3. 啟用「系統匣圖示」
4. 視桌面環境而定

## 快速擷取

### 開啟快速擷取

點擊系統匣中的 Classic 圖示：

- **macOS**：選單列圖示
- **Windows**：系統匣圖示
- **Linux**：系統匣圖示

### 使用快速擷取

1. 點擊系統匣圖示
2. 快速擷取視窗開啟
3. 輸入您的筆記
4. 按 Enter 儲存
5. 視窗自動關閉

### 快速鍵

從任何地方開啟快速擷取：

| 平台 | 快速鍵 |
|------|--------|
| **macOS** | `Cmd+Shift+N`（全域） |
| **Windows** | `Ctrl+Shift+N`（全域） |
| **Linux** | `Ctrl+Shift+N`（全域） |

## 系統匣選單

右鍵點擊系統匣圖示以查看選項：

### 選單選項

| 選項 | 操作 |
|------|------|
| **新增筆記** | 開啟快速擷取 |
| **新增每日筆記** | 開啟今天的每日筆記 |
| **搜尋** | 開啟搜尋視窗 |
| **開啟 Classic** | 開啟主視窗 |
| **設定** | 開啟設定 |
| **結束** | 關閉 Classic |

## 系統匣設定

### 在設定中配置

| 設定 | 說明 |
|------|------|
| **顯示圖示** | 切換系統匣圖示可見性 |
| **啟動時最小化** | 僅在系統匣中啟動 Classic |
| **關閉時最小化到系統匣** | 關閉視窗時繼續執行 |
| **通知** | 啟用/停用通知 |

### 登入時啟動

讓 Classic 自動啟動：

1. 開啟設定
2. 前往一般
3. 啟用「登入時啟動」
4. Classic 會在開機時啟動

## 通知

### 啟用通知

1. 開啟設定
2. 前往通知
3. 切換「啟用通知」

### 通知類型

- **提醒**：排定的提醒
- **每日筆記**：每日筆記提示
- **更新**：有可用的應用程式更新

## 視窗行為

### 關閉行為

選擇關閉視窗時發生的情況：

- **最小化到系統匣**：繼續執行，隱藏視窗
- **結束**：關閉應用程式
- **詢問**：每次提示

### 最小化行為

- **最小化到系統匣**：隱藏在系統匣中
- **最小化到工作列**：正常最小化

## 提示

### 快速擷取

使用全域快速鍵進行最快的擷取：

1. 在任何地方按 `Cmd/Ctrl+Shift+N`
2. 輸入您的想法
3. 按 Enter
4. 繼續您的工作

### 快速存取

點擊系統匣圖示以查看：

- 最近的筆記
- 快速操作
- 今天的每日筆記

### 背景執行

保持 Classic 執行以獲得：

- 更快的啟動
- 快速擷取
- 通知
- 自動備份

## 疑難排解

### 系統匣圖示遺失

**macOS**

1. 檢查圖示是否被隱藏
2. 查看控制中心（macOS Big Sur+）
3. 重新啟動 Classic

**Windows**

1. 檢查隱藏的圖示（工作列中的箭頭）
2. 將 Classic 圖示拖曳到可見區域
3. 重新啟動 Classic

**Linux**

1. 檢查桌面環境支援
2. 如需要請安裝系統匣擴充功能
3. 重新啟動 Classic

### 全域快速鍵無效

- 檢查與其他應用程式的衝突
- 在設定中重新啟用快速鍵
- 重新啟動 Classic
- 檢查作業系統權限

### 通知未顯示

**macOS**

1. 系統偏好設定 → 通知
2. 找到 Classic
3. 啟用通知

**Windows**

1. 設定 → 系統 → 通知
2. 啟用 Classic 通知

**Linux**

1. 檢查通知守護程序
2. 如需要請安裝 libnotify

### 記憶體/CPU 使用量過高

如果系統匣圖示使用過多資源：

1. 在設定中停用動畫
2. 減少通知頻率
3. 檢查應用程式更新
