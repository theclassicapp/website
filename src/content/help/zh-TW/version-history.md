# 版本歷史

Classic 會追蹤您筆記的每次變更。隨時瀏覽、比較和還原先前的版本。

## 版本歷史如何運作

### 自動追蹤

Classic 會自動儲存您筆記的版本：

- **每次編輯**：每次儲存都會建立一個版本
- **時間戳記**：知道變更發生的時間
- **完整內容**：保留完整的快照

### 保留

| 設定 | 預設值 |
|------|--------|
| **歷史保留** | 90 天 |
| **每筆記最大版本數** | 100 |
| **自動清理** | 是 |

在設定 → 編輯器 → 版本歷史中配置這些選項。

## 查看歷史

### 開啟版本歷史

1. 開啟任何筆記
2. 點擊工具列中的歷史圖示
3. 或使用 `Cmd/Ctrl+H`

### 歷史面板

歷史面板顯示：

- **時間軸**：所有版本的列表
- **時間戳記**：每個版本儲存的時間
- **預覽**：查看所選版本的內容

### 導航版本

1. 點擊列表中的任何版本
2. 預覽顯示該版本
3. 捲動查看變更
4. 與當前版本比較

## 比較版本

### 並排檢視

查看版本之間的差異：

1. 開啟版本歷史
2. 選擇一個版本
3. 點擊「比較」
4. 查看新增（綠色）和刪除（紅色）

### 差異檢視

文字差異會被高亮顯示：

- **綠色**：新增的內容
- **紅色**：移除的內容
- **未變更**：一般文字

## 還原版本

### 還原整個筆記

用先前的版本取代當前內容：

1. 開啟版本歷史
2. 選擇版本
3. 點擊「還原」
4. 確認操作

### 還原選取內容

僅複製先前版本的部分內容：

1. 開啟版本歷史
2. 選擇版本
3. 高亮您想要的文字
4. 點擊「複製到剪貼簿」
5. 貼到當前筆記中

### 從版本建立新筆記

保留兩個版本：

1. 開啟版本歷史
2. 選擇版本
3. 點擊「從版本建立筆記」
4. 新筆記即建立

## 版本歷史操作

| 操作 | 說明 |
|------|------|
| **查看** | 預覽版本 |
| **比較** | 與當前版本比較差異 |
| **還原** | 用此版本取代當前版本 |
| **複製** | 複製內容到剪貼簿 |
| **刪除** | 移除此版本 |

## 快速鍵

| 操作 | 快速鍵 |
|------|--------|
| 開啟歷史 | `Cmd/Ctrl+H` |
| 上一個版本 | `Up` |
| 下一個版本 | `Down` |
| 還原 | `Cmd/Ctrl+R` |
| 關閉歷史 | `Escape` |

## 設定

### 在設定 → 編輯器 → 版本歷史中配置

| 設定 | 選項 | 預設值 |
|------|------|--------|
| 啟用歷史 | 開/關 | 開 |
| 保留期間 | 30/60/90/180 天 | 90 天 |
| 最大版本數 | 50/100/200/500 | 100 |
| 自動儲存間隔 | 1/5/10/30 分鐘 | 5 分鐘 |

### 儲存位置

版本歷史儲存在本地：

- **macOS**：`~/Library/Application Support/Classic/history/`
- **Windows**：`%APPDATA%/Classic/history/`
- **Linux**：`~/.config/Classic/history/`

## 使用案例

### 復原意外變更

1. 開啟版本歷史
2. 找到錯誤前的版本
3. 還原該版本

### 比較草稿

1. 開啟歷史
2. 比較不同版本
3. 查看筆記如何演變

### 尋找已刪除的內容

1. 瀏覽較舊的版本
2. 搜尋已刪除的文字
3. 將其複製回來

### 追蹤進度

查看筆記隨時間的發展：

- 何時加入想法
- 結構如何變化
- 寫作演變

## 提示

### 定期檢視

定期檢視版本歷史：

- 清理不必要的版本
- 注意內容如何演變
- 從編輯模式中學習

### 在重大變更前

建立檢查點：

1. 進行小編輯
2. 儲存（建立版本）
3. 現在進行重大變更
4. 如需要可輕鬆還原

### 為重要版本命名

在筆記中加入標記：

```markdown
<!-- VERSION: 最終草稿 -->
<!-- VERSION: 重組前 -->
```

這些有助於之後識別版本。

## 疑難排解

### 歷史未儲存

- 檢查版本歷史是否已啟用
- 確認儲存位置有空間
- 檢查檔案權限

### 找不到舊版本

- 版本可能已被自動清理
- 檢查保留設定
- 增加保留期間

### 歷史佔用太多空間

1. 減少保留期間
2. 降低每筆記最大版本數
3. 執行手動清理

### 歷史載入緩慢

有許多版本的大型筆記可能載入較慢：

- 對大型歷史要有耐心
- 考慮減少最大版本數
- 封存舊筆記

## 隱私與安全

### 本地儲存

版本歷史儲存在本地：

- 永遠不會發送到伺服器
- 您的資料保持私密
- 只有您可以存取

### 加密備份

如果您使用加密備份：

- 版本歷史會被包含
- 受備份密碼保護
- 即使在備份檔案中也安全
