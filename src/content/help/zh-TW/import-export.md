# 匯入與匯出

將筆記從其他應用程式匯入 Classic，或匯出以分享或備份。Classic 支援多種格式以實現無縫資料遷移。

## 匯入筆記

### 支援的格式

| 格式 | 副檔名 | 備註 |
| ---- | ------ | ---- |
| Markdown | `.md` | ✅ 完整支援 |
| 純文字 | `.txt` | ✅ 匯入為筆記 |
| JSON | `.json` | ✅ Classic 格式 |
| HTML | `.html` | ⚠️ 轉換為 Markdown |
| Evernote | `.enex` | ⚠️ 實驗性 |
| Notion | Notion 匯出 | ⚠️ 實驗性 |
| Obsidian | Markdown 匯出 | ✅ 完整支援 |

### 匯入程序

1. 開啟 **檔案** → **匯入**（或 `Cmd/Ctrl+I`）
2. 選擇要匯入的檔案
3. 選擇目標資料夾
4. 檢視匯入預覽
5. 確認匯入

### 從剪貼簿匯入

從任何來源快速匯入：

1. 複製文字到剪貼簿
2. 開啟命令面板
3. 輸入「從剪貼簿匯入」
4. 自動建立新筆記

### 批次匯入

一次匯入多個檔案：

1. 開啟 **檔案** → **匯入**
2. 選擇 **匯入資料夾**
3. 選擇包含筆記的資料夾
4. 所有支援的檔案都會被匯入

## 匯出筆記

### 匯出格式

| 格式 | 用途 |
| ---- | ---- |
| **Markdown** | 分享、備份、其他應用程式 |
| **HTML** | 網頁發布 |
| **PDF** | 封存、列印 |
| **純文字** | 簡單分享 |
| **JSON** | API 整合 |

### 匯出單一筆記

1. 開啟筆記
2. 點擊 **檔案** → **匯出**
3. 選擇格式
4. 選擇位置
5. 儲存

### 匯出多個筆記

1. 在側邊欄中選擇筆記（Cmd/Ctrl+點擊）
2. 右鍵點擊 → **匯出所選**
3. 選擇格式
4. 選擇位置
5. 儲存

### 匯出所有筆記

1. 開啟 **檔案** → **匯出全部**
2. 選擇格式
3. 選擇位置
4. 等待匯出完成

## 遷移指南

### 從 Obsidian

Classic 可以直接匯入 Obsidian vault：

1. 開啟 **檔案** → **匯入**
2. 選擇 **Obsidian Vault**
3. 導航到您的 vault 資料夾
4. 選擇要匯入

**會轉移的內容**

- ✅ 筆記和資料夾
- ✅ 標籤
- ✅ 連結（轉換為反向連結）
- ✅ 元資料
- ⚠️ 外掛程式（不支援）

### 從 Notion

1. 從 Notion 匯出為 Markdown
2. 匯入到 Classic
3. 檢視格式
4. 修復任何損壞的連結

**Notion 遷移提示**

- 使用 Notion 的「匯出為 Markdown」
- 檢查核取方塊格式
- 確認資料庫參考
- 手動重新建立連結資料庫

### 從 Evernote

1. 從 Evernote 匯出為 HTML
2. 使用 Classic 的 HTML 匯入
3. 檢視轉換後的內容

**Evernote 遷移提示**

- 筆記本會變成資料夾
- 標籤會被保留
- 附件會被匯入
- 檢查格式

## 最佳實踐

### 匯入前

- 備份現有筆記
- 檢視匯入預覽
- 檢查是否有重複
- 確認格式

### 匯入後

- 檢視匯入的筆記
- 修復任何格式問題
- 更新內部連結
- 視需要加入標籤

### 定期匯出

- 每月匯出備份
- 針對不同目的使用不同格式
- 按日期組織匯出
- 儲存在雲端以進行異地備份

## 疑難排解

### 匯入失敗

**常見原因**

- 不支援的格式
- 檔案損壞
- 權限問題
- 記憶體限制

**解決方案**

1. 嘗試不同的格式
2. 檢查檔案完整性
3. 確認讀取權限
4. 一次匯入較少的檔案

### 格式遺失

**常見原因**

- 複雜的格式
- 不支援的語法
- 轉換限制

**解決方案**

1. 從來源應用程式使用 Markdown 匯出
2. 匯入前簡化格式
3. 匯入後手動修復

### 附件遺失

**常見原因**

- 匯入未包含附件
- 錯誤的附件路徑
- 大小限制

**解決方案**

1. 重新匯入並包含附件
2. 檢查附件資料夾
3. 單獨匯入附件
