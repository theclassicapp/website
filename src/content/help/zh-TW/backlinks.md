# 反向連結

反向連結顯示筆記之間的關聯，建立相關想法的網絡。發現您不知道存在的關係。

## 什麼是反向連結？

反向連結顯示哪些筆記連結到當前筆記。它們是您知識圖譜中的連結。

### 優點

- **發現關聯**：自動找到相關筆記
- **情境**：了解想法如何相關
- **導航**：在連結的筆記之間跳轉
- **知識圖譜**：視覺化您的思考

## 反向連結如何運作

當您建立到另一個筆記的連結時，Classic 會自動建立反向連結：

```markdown
在筆記 A 中：
詳見 [[專案規劃]]。

結果：
- 筆記 A 連結到「專案規劃」
- 「專案規劃」顯示到筆記 A 的反向連結
```

## 查看反向連結

### 反向連結面板

開啟反向連結面板以查看所有參考：

1. 開啟任何筆記
2. 點擊側邊欄中的反向連結圖示
3. 查看所有連結到此筆記的筆記

### 行內反向連結

反向連結會出現在每個筆記的底部：

- 顯示來源筆記名稱
- 顯示情境片段
- 點擊以導航

## 建立連結

### Wiki 連結語法

使用雙中括號連結到筆記：

```markdown
[[筆記名稱]]
[[另一個筆記]]
```

### 顯示文字

自訂連結文字：

```markdown
[[專案 Alpha|alpha 專案]]
[[會議筆記|昨天的會議]]
```

### 連結到標題

連結到特定章節：

```markdown
[[筆記名稱#標題]]
[[筆記名稱#章節|跳到章節]]
```

## 反向連結類型

### 直接連結

您明確建立的 Wiki 連結：

```markdown
相關：[[相似筆記]]
```

### 未連結提及

提到標題但未連結的筆記：

```markdown
筆記提到「專案 Alpha」但未連結
```

Classic 會建議將這些轉換為連結。

## 反向連結面板

### 功能

- **篩選**：搜尋反向連結
- **排序**：依日期、名稱或相關性
- **預覽**：懸停以預覽內容
- **情境**：查看周圍文字

### 操作

- 點擊以開啟連結的筆記
- 右鍵點擊查看選項
- 轉換未連結的提及

## 使用案例

### 知識管理

建立個人 Wiki：

```markdown
# 機器學習

相關於 [[人工智慧]]
# 建立在 [[統計學]] 之上
# 用於 [[自然語言處理]]
```

### 專案文件

連結相關文件：

```markdown
# API 設計

參考 [[資料庫結構]]
# 依賴於 [[認證流程]]
# 另見 [[前端整合]]
```

### 研究

連結概念和來源：

```markdown
# 研究筆記

基於 [[來源論文 A]]
# 與 [[來源論文 B]] 矛盾
# 支持 [[假設 1]]
```

### 會議筆記

連結會議與專案：

```markdown
# 會議：Q4 規劃

專案：[[專案 Alpha]]
# 上一次：[[Q3 檢討]]
# 下一次：[[Q4 啟動]]
```

## 圖譜檢視

將反向連結視覺化為網絡：

1. 開啟圖譜檢視（`Cmd/Ctrl+G`）
2. 將所有筆記視為節點
3. 線條顯示連結
4. 點擊節點以導航

### 圖譜控制

- **縮放**：捲動以縮放
- **平移**：拖曳以移動
- **篩選**：依標籤顯示/隱藏
- **聚焦**：高亮筆記的連結

## 提示

### 慷慨連結

每當引用另一個概念時就建立連結：

```markdown
✅ 良好：
討論了 [[Q4 路線圖]] 和 [[預算分配]]。

❌ 避免：
討論了 Q4 路線圖和預算分配。
```

### 使用一致的名稱

完全匹配筆記標題以確保連結可靠：

```markdown
✅ [[會議筆記]]
❌ [[會議筆記]]（大小寫有影響）
```

### 檢視反向連結

定期檢查反向連結以：

- 發現遺忘的關聯
- 尋找可合併的筆記
- 識別知識缺口

### 連結預覽

懸停在連結上以預覽而不開啟：

- 查看前幾行
- 確認是否為正確筆記
- 快速參考

## 快速鍵

| 操作 | 快速鍵 |
|------|--------|
| 開啟反向連結 | `Cmd/Ctrl+Shift+B` |
| 建立連結 | `[[`（自動完成） |
| 開啟圖譜檢視 | `Cmd/Ctrl+G` |
| 導航連結 | `Tab` 切換連結 |

## 進階功能

### 嵌入

嵌入其他筆記的內容：

```markdown
![[筆記名稱]]
![[筆記名稱#章節]]
```

### 轉錄

以行內方式包含內容：

```markdown
## 相關內容
![[共享範本#標題]]
```

### 連結著色

連結根據以下情況變色：

- **藍色**：未訪問的連結
- **紫色**：已訪問的連結
- **紅色**：損壞的連結（筆記不存在）

## 疑難排解

### 反向連結遺失

- 檢查連結語法 `[[筆記名稱]]`
- 確認筆記標題完全匹配
- 重新整理反向連結面板

### 連結損壞

- 筆記可能已重新命名
- 筆記可能已刪除
- 使用搜尋找到正確的筆記

### 重複連結

- 整合相似筆記
- 對重新命名的筆記使用重新導向
- 檢查標題變化
