# ローカルファーストアーキテクチャ

Classicはプライバシーを基盤として構築されています。データは常にデバイス上に留まります。

## 「ローカルファースト」とは？

ローカルファーストとは：

1. **データはローカルに保存** - すべてのノート、添付ファイル、設定はコンピューターのフォルダーに保存されます。
2. **クラウド不要** - Classicはインターネット接続なしで動作します。
3. **データはあなたのもの** - データを完全にコントロールできます。

## データストレージ

### デフォルトの場所

データは専用フォルダーに保存されます：

| プラットフォーム | 場所 |
| -------- | -------- |
| **macOS** | `~/Library/Application Support/Classic/notes` |
| **Windows** | `%APPDATA%\Classic\notes` |
| **Linux** | `~/.config/Classic/notes` |

### データ形式

- ノートはプレーンテキストファイル（`.md`）として保存
- 添付ファイルは `attachments` サブフォルダーに保存
- 設定は `settings.json` に保存

### ファイル構造

```
notes/
├── 受信トレイ/
├── デイリーノート/
│   └── 2024-01-15.md
├── プロジェクト/
│   ├── マイプロジェクト.md
│   └── サブノート.md
└── attachments/
    └── image.png
settings.json
```

## プライバシーの利点

### アカウント不要

- サインアップ不要
- メール確認不要
- 個人情報の収集なし

### トラッキングなし

- アナリティクスなし
- テレメトリーなし
- 使用データの外部サーバーへの送信なし

### オフライン対応

- インターネットなしで動作
- 選択した場合のみ同期
- データは常にアクセス可能

## データセキュリティ

### あなたの責任

- **バックアップ**: 設定で自動バックアップを設定
- **エクスポート**: 定期的にデータをエクスポート
- **更新**: セキュリティ修正のためにClassicを更新

### 私たちの責任

- **ローカルストレージ**: データを見たり触れたりしません
- **バックドアなし**: 隠れたデータ収集なし
- **透明性**: データ取り扱いについてオープンで誠実

## 同期（オプション）

Classicには組み込みのクラウド同期は含まれていません。ただし、データを同期できます：

### iCloud/OneDrive/Dropboxを使用

1. Classicの設定を開く
2. ノートフォルダーの場所をクラウドフォルダーに変更
3. Classicはクラウドプロバイダーを通じて同期

### Gitを使用

1. ノートフォルダーでgitリポジトリを初期化
2. 手動または自動化で変更をコミットしてプッシュ

## データ復旧

### バックアップから

1. 設定を開く
2. バックアップと復元に移動
3. バックアップファイルを選択
4. 復元をクリック

### 手動復旧

1. ノートフォルダーを見つける
2. ファイルを安全な場所にコピー
3. 新しいClassicインストールにインポート

## よくある質問

### データは本当にプライベートですか？

**はい。** サードパーティのサービスを使用して明示的に同期しない限り、データはデバイスを離れません。

### データを別のコンピューターに移動できますか？

**はい。** ノートフォルダーを新しいコンピューターにコピーし、設定でClassicに指定するだけです。

### Classicをアンインストールするとどうなりますか？

**データは残ります。** Classicをアンインストールしてもノートフォルダーは削除されません。データを失うことなく安全にアンインストールと再インストールが可能です。

### Classicはデータにアクセスできますか？

**いいえ。** Classicはデータ収集のための外部接続のないローカルアプリケーションです。
