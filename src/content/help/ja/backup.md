# バックアップと復元

Classicの組み込みバックアップシステムでノートを安全に保ちましょう。二度と作業を失うことはありません。

## 自動バックアップ

### 仕組み

Classicは定期的にノートのバックアップを自動的に作成

- **デフォルトの頻度**: 30分ごと
- **バックアップ場所**: ノートと同じフォルダー
- **保持期間**: 最後の10バックアップを維持
- **形式**: 圧縮アーカイブ（`.cbak`）

### 自動バックアップの設定

1. 設定を開く
2. **バックアップと復元**に移動
3. 以下を設定

| 設定 | オプション | デフォルト |
| ------- | ------- | ------- |
| 自動バックアップを有効化 | オン/オフ | オン |
| バックアップ頻度 | 15分 / 30分 / 1時間 / 4時間 | 30分 |
| バックアップを保持 | 5 / 10 / 20 / 50 | 10 |
| バックアップ場所 | カスタムパス | ノートフォルダー |

## 手動バックアップ

### 今すぐバックアップを作成

1. コマンドパレットを開く（`Cmd/Ctrl+P`）
2. 「バックアップ」と入力
3. **今すぐバックアップを作成**を選択
4. 場所を選択（オプション）
5. バックアップが作成されます

### バックアップファイル形式

バックアップは `.cbak` ファイルとして保存され、以下が含まれます

- すべてのノート（Markdownファイル）
- 添付ファイル
- 設定
- メタデータ（タグ、リンクなど）

## バックアップから復元

### 復元プロセス

1. 設定を開く
2. **バックアップと復元**に移動
3. **バックアップから復元**をクリック
4. バックアップファイルを選択
5. 復元オプションを選択
6. 復元を確認

### 復元オプション

| オプション | 説明 |
| ------ | ----------- |
| **すべて置換** | 現在のノートを完全にバックアップで置換 |
| **マージ** | バックアップを既存のノートと結合 |
| **プレビュー** | 復元前にバックアップの内容を表示 |

### 復元されるもの

- ✅ すべてのノートとフォルダー
- ✅ 添付ファイル
- ✅ タグとメタデータ
- ✅ ノート履歴
- ⚠️ 設定（オプション）
- ❌ アプリケーション設定

## バックアップの場所

### デフォルトの場所

```
~/Classic/backups/
├── backup-2024-01-15-10-30.cbak
├── backup-2024-01-15-11-00.cbak
└── backup-2024-01-15-11-30.cbak
```

### カスタムの場所

1. 設定を開く
2. バックアップと復元に移動
3. 「バックアップ場所を変更」をクリック
4. 好みのフォルダーを選択

### クラウドバックアップ

バックアップ場所をクラウド同期フォルダーに指定

- **iCloud**: `~/Library/Mobile Documents/com~apple~CloudDocs/Classic Backups`
- **Dropbox**: `~/Dropbox/Classic Backups`
- **Google Drive**: `~/Google Drive/Classic Backups`
- **OneDrive**: `~/OneDrive/Classic Backups`

## ベストプラクティス

### バックアップスケジュール

- **自動バックアップ**: 30分間隔で有効にしておく
- **手動バックアップ**: 大きな変更の前に作成
- **オフサイトバックアップ**: 定期的にバックアップを外部ストレージにコピー

### バックアップの検証

定期的にバックアップを検証

1. テスト復元を作成
2. ノートの内容を確認
3. 添付ファイルを検証
4. メタデータを確認

### バックアップローテーション

- 最近のバックアップはローカルに保持
- 古いバックアップは外部ストレージにアーカイブ
- 非常に古いバックアップは削除してスペースを節約

## トラブルシューティング

### バックアップ失敗

**考えられる原因**

- ディスク容量不足
- 権限の問題
- ファイルが使用中

**解決策**

1. 利用可能なディスク容量を確認
2. フォルダーの権限を確認
3. 他のアプリケーションを閉じる
4. 別の場所を試す

### 復元失敗

**考えられる原因**

- 破損したバックアップファイル
- 互換性のないバージョン
- 容量不足

**解決策**

1. 古いバックアップを試す
2. バックアップファイルの整合性を確認
3. ディスク容量を確保
4. サポートに連絡

### バックアップが見つからない

**考えられる原因**

- 自動バックアップが無効
- バックアップ場所が変更された
- バックアップが削除された

**解決策**

1. バックアップ設定を確認
2. バックアップ場所を確認
3. 自動バックアップを有効化
4. 今すぐ手動バックアップを作成

## 高度なオプション

### 暗号化バックアップ

追加のセキュリティのため

1. 設定を開く
2. バックアップと復元に移動
3. 「バックアップを暗号化」を有効化
4. パスワードを設定
5. このパスワードを覚えておく - 復元できません

### 増分バックアップ

増分バックアップでスペースを節約

- 最後のバックアップ以降の変更のみを保存
- 完全復元で完全な状態を再構築
- バックアップ時間とサイズを削減

### バックアップ除外

特定のコンテンツをバックアップから除外

1. 設定を開く
2. バックアップと復元に移動
3. 「除外」をクリック
4. 除外するフォルダーまたはファイルを追加
