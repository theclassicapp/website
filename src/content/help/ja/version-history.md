# バージョン履歴

Classicはノートへのすべての変更を追跡します。必要なときにいつでも以前のバージョンをブラウズ、比較、復元できます。

## バージョン履歴の仕組み

### 自動追跡

Classicはノートのバージョンを自動的に保存

- **すべての編集**: 各保存がバージョンを作成
- **タイムスタンプ付き**: 変更がいつ発生したかを把握
- **完全なコンテンツ**: 完全なスナップショットを保存

### 保持

| 設定 | デフォルト |
|---------|---------|
| **履歴保持期間** | 90日 |
| **最大バージョン数** | ノートあたり100 |
| **自動クリーンアップ** | あり |

これらは設定 → エディター → バージョン履歴で構成します。

## 履歴の表示

### バージョン履歴を開く

1. 任意のノートを開く
2. ツールバーの履歴アイコンをクリック
3. または `Cmd/Ctrl+H` を使用

### 履歴パネル

履歴パネルには以下が表示

- **タイムライン**: すべてのバージョンのリスト
- **タイムスタンプ**: 各バージョンが保存された時刻
- **プレビュー**: 選択したバージョンのコンテンツを表示

### バージョンのナビゲート

1. リストの任意のバージョンをクリック
2. プレビューにそのバージョンが表示
3. 変更をスクロール
4. 現在のバージョンと比較

## バージョンの比較

### サイドバイサイドビュー

バージョン間の違いを表示

1. バージョン履歴を開く
2. バージョンを選択
3. 「比較」をクリック
4. 追加（緑）と削除（赤）を表示

### 差分ビュー

テキストの違いがハイライト

- **緑**: 追加されたコンテンツ
- **赤**: 削除されたコンテンツ
- **変更なし**: 通常のテキスト

## バージョンの復元

### ノート全体を復元

現在のコンテンツを以前のバージョンで置換

1. バージョン履歴を開く
2. バージョンを選択
3. 「復元」をクリック
4. アクションを確認

### 選択部分を復元

以前のバージョンの一部だけをコピー

1. バージョン履歴を開く
2. バージョンを選択
3. 必要なテキストをハイライト
4. 「クリップボードにコピー」をクリック
5. 現在のノートに貼り付け

### バージョンから新規ノートを作成

両方のバージョンを保持

1. バージョン履歴を開く
2. バージョンを選択
3. 「バージョンからノートを作成」をクリック
4. 新しいノートが作成されます

## バージョン履歴のアクション

| アクション | 説明 |
|--------|-------------|
| **表示** | バージョンをプレビュー |
| **比較** | 現在と比較 |
| **復元** | 現在をこれで置換 |
| **コピー** | コンテンツをクリップボードにコピー |
| **削除** | このバージョンを削除 |

## キーボードショートカット

| アクション | ショートカット |
|--------|----------|
| 履歴を開く | `Cmd/Ctrl+H` |
| 前のバージョン | `Up` |
| 次のバージョン | `Down` |
| 復元 | `Cmd/Ctrl+R` |
| 履歴を閉じる | `Escape` |

## 設定

### 設定 → エディター → バージョン履歴で構成

| 設定 | オプション | デフォルト |
|---------|---------|---------|
| 履歴を有効化 | オン/オフ | オン |
| 保持期間 | 30/60/90/180日 | 90日 |
| 最大バージョン数 | 50/100/200/500 | 100 |
| 自動保存間隔 | 1/5/10/30分 | 5分 |

### ストレージ場所

バージョン履歴はローカルに保存

- **macOS**: `~/Library/Application Support/Classic/history/`
- **Windows**: `%APPDATA%/Classic/history/`
- **Linux**: `~/.config/Classic/history/`

## 使用例

### 誤った変更の復旧

1. バージョン履歴を開く
2. ミスの前のバージョンを見つける
3. そのバージョンを復元

### ドラフトの比較

1. 履歴を開く
2. 異なるバージョンを比較
3. ノートがどのように進化したかを表示

### 削除されたコンテンツを見つける

1. 古いバージョンをブラウズ
2. 削除されたテキストを検索
3. コピーして戻す

### 進捗の追跡

ノートが時間とともにどのように発展したかを表示

- アイデアがいつ追加されたか
- 構造がどのように変わったか
- 執筆の進化

## ヒント

### 定期的なレビュー

定期的にバージョン履歴をレビュー

- 不要なバージョンをクリーンアップ
- コンテンツがどのように進化したかをメモ
- 編集パターンから学ぶ

### 大きな変更の前に

チェックポイントを作成

1. 小さな編集を行う
2. 保存（バージョンを作成）
3. 大きな変更を行う
4. 必要に応じて簡単に復元

### 重要なバージョンに名前を付ける

ノートにマーカーを追加

```markdown
<!-- VERSION: 最終ドラフト -->
<!-- VERSION: 再構築前 -->
```

これらは後でバージョンを特定するのに役立ちます。

## トラブルシューティング

### 履歴が保存されない

- バージョン履歴が有効かどうか確認
- ストレージ場所に容量があるか確認
- ファイル権限を確認

### 古いバージョンが見つからない

- バージョンが自動クリーンアップされた可能性
- 保持設定を確認
- 保持期間を延長

### 履歴がスペースを使いすぎている

1. 保持期間を短縮
2. ノートあたりの最大バージョン数を減らす
3. 手動クリーンアップを実行

### 履歴の読み込みが遅い

多くのバージョンを持つ大きなノートは読み込みが遅い可能性

- 大きな履歴には忍耐が必要
- 最大バージョン数を減らすことを検討
- 古いノートをアーカイブ

## プライバシーとセキュリティ

### ローカルストレージ

バージョン履歴はローカルに保存

- サーバーに送信されない
- データはプライベート
- あなただけがアクセス可能

### 暗号化バックアップ

暗号化バックアップを使用する場合

- バージョン履歴が含まれる
- バックアップパスワードで保護
- バックアップファイルでも安全
