# インポートとエクスポート

他のアプリからノートをClassicに取り込む、または共有やバックアップのためにエクスポート。Classicはシームレスなデータ移行のために複数の形式をサポートしています。

## ノートのインポート

### サポートされている形式

| 形式 | 拡張子 | ノート |
| ------ | --------- | ----- |
| Markdown | `.md` | ✅ 完全サポート |
| プレーンテキスト | `.txt` | ✅ ノートとしてインポート |
| JSON | `.json` | ✅ Classic形式 |
| HTML | `.html` | ⚠️ Markdownに変換 |
| Evernote | `.enex` | ⚠️ 実験的 |
| Notion | Notionエクスポート | ⚠️ 実験的 |
| Obsidian | Markdownエクスポート | ✅ 完全サポート |

### インポートプロセス

1. **ファイル** → **インポート**を開く（または `Cmd/Ctrl+I`）
2. インポートするファイルを選択
3. 保存先フォルダーを選択
4. インポートプレビューを確認
5. インポートを確認

### クリップボードからインポート

任意のソースからクイックインポート

1. テキストをクリップボードにコピー
2. コマンドパレットを開く
3. 「クリップボードからインポート」と入力
4. 新しいノートが自動的に作成

### 一括インポート

複数のファイルを一度にインポート

1. **ファイル** → **インポート**を開く
2. **フォルダーをインポート**を選択
3. ノートを含むフォルダーを選択
4. サポートされているすべてのファイルがインポート

## ノートのエクスポート

### エクスポート形式

| 形式 | 使用例 |
| ------ | ------- |
| **Markdown** | 共有、バックアップ、他のアプリ |
| **HTML** | Web公開 |
| **PDF** | アーカイブ、印刷 |
| **プレーンテキスト** | シンプルな共有 |
| **JSON** | API統合 |

### 単一ノートをエクスポート

1. ノートを開く
2. **ファイル** → **エクスポート**をクリック
3. 形式を選択
4. 場所を選択
5. 保存

### 複数ノートをエクスポート

1. サイドバーでノートを選択（Cmd/Ctrl+クリック）
2. 右クリック → **選択をエクスポート**
3. 形式を選択
4. 場所を選択
5. 保存

### すべてのノートをエクスポート

1. **ファイル** → **すべてエクスポート**を開く
2. 形式を選択
3. 場所を選択
4. エクスポートの完了を待つ

## 移行ガイド

### Obsidianから

ClassicはObsidianボールトを直接インポート可能

1. **ファイル** → **インポート**を開く
2. **Obsidianボールト**を選択
3. ボールトフォルダーに移動
4. インポートを選択

**移行されるもの**

- ✅ ノートとフォルダー
- ✅ タグ
- ✅ リンク（バックリンクに変換）
- ✅ メタデータ
- ⚠️ プラグイン（サポートされていません）

### Notionから

1. NotionからMarkdownとしてエクスポート
2. Classicにインポート
3. フォーマットをレビュー
4. 壊れたリンクを修正

**Notion移行のヒント**

- Notionの「Markdownとしてエクスポート」を使用
- チェックボックスのフォーマットを確認
- データベース参照を確認
- リンクされたデータベースは手動で再作成

### Evernoteから

1. EvernoteからHTMLとしてエクスポート
2. ClassicのHTMLインポートを使用
3. 変換されたコンテンツをレビュー

**Evernote移行のヒント**

- ノートブックはフォルダーになります
- タグは保持されます
- 添付ファイルはインポートされます
- フォーマットを確認

## ベストプラクティス

### インポート前

- 既存のノートをバックアップ
- インポートプレビューを確認
- 重複を確認
- フォーマットを確認

### インポート後

- インポートされたノートをレビュー
- フォーマットの問題を修正
- 内部リンクを更新
- 必要に応じてタグを追加

### 定期的なエクスポート

- 月次でバックアップをエクスポート
- 異なる目的で異なる形式を使用
- エクスポートを日付で整理
- オフサイトバックアップのためにクラウドに保存

## トラブルシューティング

### インポート失敗

**一般的な原因**

- サポートされていない形式
- 破損したファイル
- 権限の問題
- メモリの制限

**解決策**

1. 別の形式を試す
2. ファイルの整合性を確認
3. 読み取り権限を確認
4. 一度に少ないファイルをインポート

### フォーマットが失われた

**一般的な原因**

- 複雑なフォーマット
- サポートされていない構文
- 変換の制限

**解決策**

1. ソースアプリからMarkdownエクスポートを使用
2. インポート前にフォーマットを簡素化
3. インポート後に手動で修正

### 添付ファイルが見つからない

**一般的な原因**

- インポートに添付ファイルが含まれていない
- 添付ファイルのパスが間違っている
- サイズの制限

**解決策**

1. 添付ファイル付きで再インポート
2. 添付ファイルフォルダーを確認
3. 添付ファイルを個別にインポート
