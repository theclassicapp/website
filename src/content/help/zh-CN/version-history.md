# 版本历史

Classic 会追踪你笔记的每一次更改。随时浏览、比较和恢复以前的版本。

## 版本历史如何工作

### 自动追踪

Classic 自动保存你的笔记版本

- **每次编辑**：每次保存创建一个版本
- **时间戳**：知道更改发生的时间
- **完整内容**：保留完整快照

### 保留期限

| 设置 | 默认值 |
|---------|---------|
| **历史保留期** | 90 天 |
| **最大版本数** | 每个笔记 100 个 |
| **自动清理** | 是 |

在设置 → 编辑器 → 版本历史中配置这些选项。

## 查看历史

### 打开版本历史

1. 打开任意笔记
2. 点击工具栏中的历史图标
3. 或使用 `Cmd/Ctrl+H`

### 历史面板

历史面板显示

- **时间线**：所有版本列表
- **时间戳**：每个版本的保存时间
- **预览**：查看所选版本的内容

### 导航版本

1. 点击列表中的任意版本
2. 预览显示该版本
3. 滚动查看更改
4. 与当前版本比较

## 比较版本

### 并排视图

查看版本之间的差异

1. 打开版本历史
2. 选择一个版本
3. 点击"比较"
4. 查看添加（绿色）和删除（红色）

### 差异视图

文本差异被高亮显示

- **绿色**：添加的内容
- **红色**：删除的内容
- **未更改**：普通文本

## 恢复版本

### 恢复整个笔记

用以前的版本替换当前内容

1. 打开版本历史
2. 选择版本
3. 点击"恢复"
4. 确认操作

### 恢复选择内容

仅复制以前版本的一部分

1. 打开版本历史
2. 选择版本
3. 高亮你想要的文本
4. 点击"复制到剪贴板"
5. 粘贴到当前笔记

### 从版本创建新笔记

保留两个版本

1. 打开版本历史
2. 选择版本
3. 点击"从版本创建笔记"
4. 创建新笔记

## 版本历史操作

| 操作 | 描述 |
|--------|-------------|
| **查看** | 预览版本 |
| **比较** | 与当前版本对比 |
| **恢复** | 用此版本替换当前版本 |
| **复制** | 复制内容到剪贴板 |
| **删除** | 删除此版本 |

## 键盘快捷键

| 操作 | 快捷键 |
|--------|----------|
| 打开历史 | `Cmd/Ctrl+H` |
| 上一个版本 | `Up` |
| 下一个版本 | `Down` |
| 恢复 | `Cmd/Ctrl+R` |
| 关闭历史 | `Escape` |

## 设置

### 在设置 → 编辑器 → 版本历史中配置

| 设置 | 选项 | 默认值 |
|---------|---------|---------|
| 启用历史 | 开/关 | 开 |
| 保留期限 | 30/60/90/180 天 | 90 天 |
| 最大版本数 | 50/100/200/500 | 100 |
| 自动保存间隔 | 1/5/10/30 分钟 | 5 分钟 |

### 存储位置

版本历史存储在本地

- **macOS**: `~/Library/Application Support/Classic/history/`
- **Windows**: `%APPDATA%/Classic/history/`
- **Linux**: `~/.config/Classic/history/`

## 使用案例

### 恢复意外更改

1. 打开版本历史
2. 找到错误之前的版本
3. 恢复该版本

### 比较草稿

1. 打开历史
2. 比较不同版本
3. 查看笔记如何演变

### 查找已删除内容

1. 浏览较旧版本
2. 搜索已删除文本
3. 将其复制回来

### 追踪进度

查看笔记随时间的发展

- 何时添加想法
- 结构如何变化
- 写作演变

## 提示

### 定期审查

定期审查版本历史

- 清理不必要的版本
- 注意内容如何演变
- 从编辑模式中学习

### 在重大更改前

创建检查点

1. 进行小编辑
2. 保存（创建版本）
3. 现在进行重大更改
4. 如需要易于回滚

### 命名重要版本

在笔记中添加标记

```markdown
<!-- VERSION: 最终草稿 -->
<!-- VERSION: 重构前 -->
```

这些有助于稍后识别版本。

## 故障排除

### 历史未保存

- 检查版本历史是否启用
- 验证存储位置有空间
- 检查文件权限

### 找不到旧版本

- 版本可能已被自动清理
- 检查保留设置
- 增加保留期限

### 历史占用太多空间

1. 减少保留期限
2. 降低每个笔记的最大版本数
3. 运行手动清理

### 历史加载缓慢

有大量版本的大型笔记可能加载缓慢

- 对大型历史要有耐心
- 考虑减少最大版本数
- 归档旧笔记

## 隐私与安全

### 本地存储

版本历史存储在本地

- 从不发送到服务器
- 你的数据保持私密
- 只有你有访问权限

### 加密备份

如果你使用加密备份

- 版本历史包含在内
- 由你的备份密码保护
- 即使在备份文件中也安全
