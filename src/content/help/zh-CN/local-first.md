# 本地优先架构

Classic 以隐私为基础构建。你的数据始终保存在你的设备上。

## "本地优先"意味着什么？

本地优先意味着：

1. **你的数据存储在本地**：你的所有笔记、附件和设置都存储在电脑的文件夹中。
2. **无需云端**：Classic 无需互联网连接即可工作。
3. **你拥有你的数据**：你完全控制你的数据。

## 数据存储

### 默认位置

你的数据存储在专用文件夹中：

| 平台 | 位置 |
| -------- | -------- |
| **macOS** | `~/Library/Application Support/Classic/notes` |
| **Windows** | `%APPDATA%\Classic\notes` |
| **Linux** | `~/.config/Classic/notes` |

### 数据格式

- 笔记存储为纯文本文件（`.md`）
- 附件存储在 `attachments` 子文件夹中
- 设置存储在 `settings.json` 中

### 文件结构

```
notes/
├── 收件箱/
├── 日记/
│   └── 2024-01-15.md
├── 项目/
│   ├── 我的项目.md
│   └── 子笔记.md
└── attachments/
    └── image.png
settings.json
```

## 隐私优势

### 无需账户

- 无需注册
- 无需邮箱验证
- 不收集个人信息

### 无追踪

- 无分析
- 无遥测
- 不向外部服务器发送使用数据

### 可离线使用

- 无需互联网即可工作
- 仅在你选择时同步
- 你的数据始终可访问

## 数据安全

### 你的责任

- **备份**：在设置中设置自动备份
- **导出**：定期导出数据
- **更新**：保持 Classic 更新以获取安全修复

### 我们的责任

- **本地存储**：我们从不查看或触碰你的数据
- **无后门**：无隐藏数据收集
- **透明**：对数据处理保持开放和诚实

## 同步（可选）

Classic 不包含内置云同步。但是，你可以同步你的数据：

### 使用 iCloud/OneDrive/Dropbox

1. 打开 Classic 设置
2. 将笔记文件夹位置更改为你的云文件夹
3. Classic 将通过你的云提供商同步

### 使用 Git

1. 在笔记文件夹中初始化 git 仓库
2. 手动或通过自动化提交和推送更改

## 数据恢复

### 从备份恢复

1. 打开设置
2. 进入备份与恢复
3. 选择备份文件
4. 点击恢复

### 手动恢复

1. 找到你的笔记文件夹
2. 将文件复制到安全位置
3. 导入到新的 Classic 安装中

## 常见问题

### 我的数据真的私密吗？

**是的。** 除非你明确使用第三方服务同步，否则你的数据不会离开你的设备。

### 我可以将数据移动到另一台电脑吗？

**可以。** 只需将笔记文件夹复制到新电脑，并在设置中指向 Classic。

### 如果我卸载 Classic 会怎样？

**你的数据保留。** 卸载 Classic 不会删除你的笔记文件夹。你可以安全地卸载和重新安装而不丢失数据。

### Classic 可以访问我的数据吗？

**不可以。** Classic 是一个本地应用程序，没有用于数据收集的外部连接。
