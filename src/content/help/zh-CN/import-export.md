# 导入与导出

从其他应用将笔记导入 Classic，或导出它们以分享或备份。Classic 支持多种格式实现无缝数据迁移。

## 导入笔记

### 支持的格式

| 格式 | 扩展名 | 说明 |
| ------ | --------- | ----- |
| Markdown | `.md` | ✅ 完全支持 |
| 纯文本 | `.txt` | ✅ 作为笔记导入 |
| JSON | `.json` | ✅ Classic 格式 |
| HTML | `.html` | ⚠️ 转换为 Markdown |
| Evernote | `.enex` | ⚠️ 实验性 |
| Notion | Notion 导出 | ⚠️ 实验性 |
| Obsidian | Markdown 导出 | ✅ 完全支持 |

### 导入过程

1. 打开**文件** → **导入**（或 `Cmd/Ctrl+I`）
2. 选择要导入的文件
3. 选择目标文件夹
4. 查看导入预览
5. 确认导入

### 从剪贴板导入

从任意来源快速导入

1. 将文本复制到剪贴板
2. 打开命令面板
3. 输入"从剪贴板导入"
4. 自动创建新笔记

### 批量导入

一次导入多个文件

1. 打开**文件** → **导入**
2. 选择**导入文件夹**
3. 选择包含笔记的文件夹
4. 所有支持的文件都会被导入

## 导出笔记

### 导出格式

| 格式 | 用途 |
| ------ | ------- |
| **Markdown** | 分享、备份、其他应用 |
| **HTML** | 网页发布 |
| **PDF** | 归档、打印 |
| **纯文本** | 简单分享 |
| **JSON** | API 集成 |

### 导出单个笔记

1. 打开笔记
2. 点击**文件** → **导出**
3. 选择格式
4. 选择位置
5. 保存

### 导出多个笔记

1. 在侧边栏选择笔记（Cmd/Ctrl+点击）
2. 右键点击 → **导出所选**
3. 选择格式
4. 选择位置
5. 保存

### 导出所有笔记

1. 打开**文件** → **导出全部**
2. 选择格式
3. 选择位置
4. 等待导出完成

## 迁移指南

### 从 Obsidian

Classic 可以直接导入 Obsidian 仓库

1. 打开**文件** → **导入**
2. 选择**Obsidian 仓库**
3. 导航到你的仓库文件夹
4. 选择导入

**迁移内容**

- ✅ 笔记和文件夹
- ✅ 标签
- ✅ 链接（转换为反向链接）
- ✅ 元数据
- ⚠️ 插件（不支持）

### 从 Notion

1. 从 Notion 导出为 Markdown
2. 导入到 Classic
3. 审查格式
4. 修复任何损坏的链接

**Notion 迁移提示**

- 使用 Notion 的"导出为 Markdown"
- 检查复选框格式
- 验证数据库引用
- 手动重新创建链接数据库

### 从 Evernote

1. 从 Evernote 导出为 HTML
2. 使用 Classic 的 HTML 导入
3. 审查转换的内容

**Evernote 迁移提示**

- 笔记本变为文件夹
- 标签保留
- 附件被导入
- 检查格式

## 最佳实践

### 导入前

- 备份现有笔记
- 审查导入预览
- 检查重复项
- 验证格式

### 导入后

- 审查导入的笔记
- 修复任何格式问题
- 更新内部链接
- 根据需要添加标签

### 定期导出

- 每月导出备份
- 为不同目的使用不同格式
- 按日期组织导出
- 存储在云端以实现异地备份

## 故障排除

### 导入失败

**常见原因**

- 不支持的格式
- 文件损坏
- 权限问题
- 内存限制

**解决方案**

1. 尝试不同格式
2. 检查文件完整性
3. 验证读取权限
4. 一次导入较少文件

### 格式丢失

**常见原因**

- 复杂格式
- 不支持的语法
- 转换限制

**解决方案**

1. 从源应用使用 Markdown 导出
2. 导入前简化格式
3. 导入后手动修复

### 附件丢失

**常见原因**

- 导入未包含附件
- 附件路径错误
- 大小限制

**解决方案**

1. 带附件重新导入
2. 检查附件文件夹
3. 单独导入附件
