# สถาปัตยกรรม Local-First

Classic ถูกสร้างขึ้นโดยมีความเป็นส่วนตัวเป็นรากฐาน ข้อมูลของคุณอยู่บนอุปกรณ์ของคุณ เสมอ

## "Local-First" หมายความว่าอะไร?

Local-first หมายความว่า:

1. **ข้อมูลของคุณถูกเก็บไว้ในเครื่อง** บนคอมพิวเตอร์ของคุณ บันทึก ไฟล์แนบ และการตั้งค่าทั้งหมดของคุณถูกเก็บไว้ในโฟลเดอร์บนคอมพิวเตอร์ของคุณ
2. **ไม่จำเป็นต้องใช้คลาวด์** Classic ทำงานโดยไม่ต้องใช้อินเทอร์เน็ต
3. **คุณเป็นเจ้าของข้อมูลของคุณ** คุณมีการควบคุมข้อมูลของคุณอย่างสมบูรณ์

## การจัดเก็บข้อมูล

### ตำแหน่งเริ่มต้น

ข้อมูลของคุณถูกเก็บไว้ในโฟลเดอร์เฉพาะ:

| แพลตฟอร์ม | ตำแหน่ง |
| -------- | -------- |
| **macOS** | `~/Library/Application Support/Classic/notes` |
| **Windows** | `%APPDATA%\Classic\notes` |
| **Linux** | `~/.config/Classic/notes` |

### รูปแบบข้อมูล

- บันทึกถูกเก็บเป็นไฟล์ข้อความธรรมดา (`.md`)
- ไฟล์แนบถูกเก็บในโฟลเดอร์ย่อย `attachments`
- การตั้งค่าถูกเก็บใน `settings.json`

### โครงสร้างไฟล์

```
notes/
├── Inbox/
├── Daily Notes/
│   └── 2024-01-15.md
├── Projects/
│   ├── My Project.md
│   └── sub-note.md
└── attachments/
    └── image.png
settings.json
```

## ประโยชน์ด้านความเป็นส่วนตัว

### ไม่ต้องใช้บัญชี

- ไม่ต้องลงทะเบียน
- ไม่ต้องยืนยันอีเมล
- ไม่มีการเก็บรวบรวมข้อมูลส่วนบุคคล

### ไม่มีการติดตาม

- ไม่มี analytics
- ไม่มี telemetry
- ไม่มีข้อมูลการใช้งานถูกส่งไปยังเซิร์ฟเวอร์ภายนอก

### ทำงานแบบออฟไลน์ได้

- ทำงานโดยไม่ต้องใช้อินเทอร์เน็ต
- ซิงค์เฉพาะเมื่อคุณเลือก
- ข้อมูลของคุณสามารถเข้าถึงได้เสมอ

## ความปลอดภัยของข้อมูล

### ความรับผิดชอบของคุณ

- **การสำรองข้อมูล**: ตั้งค่าการสำรองข้อมูลอัตโนมัติในการตั้งค่า
- **การส่งออก**: ส่งออกข้อมูลของคุณอย่างสม่ำเสมอ
- **การอัปเดต**: รักษา Classic ให้เป็นปัจจุบันสำหรับการแก้ไขความปลอดภัย

### ความรับผิดชอบของเรา

- **การจัดเก็บในเครื่อง**: เราไม่เคยเห็นหรือแตะข้อมูลของคุณ
- **ไม่มีประตูหลัง**: ไม่มีการเก็บรวบรวมข้อมูลที่ซ่อนอยู่
- **ความโปร่งใส**: เปิดเผยและซื่อสัตย์เกี่ยวกับการจัดการข้อมูล

## การซิงค์ (ไม่บังคับ)

Classic ไม่รวมการซิงค์คลาวด์ในตัว อย่างไรก็ตาม คุณสามารถซิงค์ข้อมูลของคุณ:

### ใช้ iCloud/OneDrive/Dropbox

1. เปิดการตั้งค่า Classic
2. เปลี่ยนตำแหน่งโฟลเดอร์ Notes เป็นโฟลเดอร์คลาวด์ของคุณ
3. Classic จะซิงค์ผ่านผู้ให้บริการคลาวด์ของคุณ

### ใช้ Git

1. เริ่มต้น git repo ในโฟลเดอร์บันทึกของคุณ
2. Commit และ push การเปลี่ยนแปลงด้วยตนเองหรือด้วยระบบอัตโนมัติ

## การกู้คืนข้อมูล

### จากการสำรองข้อมูล

1. เปิดการตั้งค่า
2. ไปที่ Backup & Restore
3. เลือกไฟล์สำรองข้อมูล
4. คลิก Restore

### การกู้คืนด้วยตนเอง

1. ค้นหาโฟลเดอร์บันทึกของคุณ
2. คัดลอกไฟล์ไปยังตำแหน่งที่ปลอดภัย
3. นำเข้าสู่การติดตั้ง Classic ใหม่

## คำถามที่พบบ่อย

### ข้อมูลของฉันเป็นส่วนตัวจริงหรือ?

**ใช่** ข้อมูลของคุณไม่เคยออกจากอุปกรณ์ของคุณ เว้นแต่คุณจะซิงค์โดยชัดเจนโดยใช้บริการบุคคลที่สาม

### ฉันสามารถย้ายข้อมูลไปยังคอมพิวเตอร์เครื่องอื่นได้ไหม?

**ได้** เพียงคัดลอกโฟลเดอร์บันทึกของคุณไปยังคอมพิวเตอร์เครื่องใหม่และชี้ Classic ไปที่มันในการตั้งค่า

### จะเกิดอะไรขึ้นถ้าฉันถอนการติดตั้ง Classic?

**ข้อมูลของคุณยังคงอยู่** การถอนการติดตั้ง Classic ไม่ได้ลบโฟลเดอร์บันทึกของคุณ คุณสามารถถอนการติดตั้งและติดตั้งใหม่ได้อย่างปลอดภัยโดยไม่สูญเสียข้อมูล

### Classic สามารถเข้าถึงข้อมูลของฉันได้ไหม?

**ไม่ได้** Classic เป็นแอปพลิเคชันในเครื่องที่ไม่มีการเชื่อมต่อภายนอกสำหรับการเก็บรวบรวมข้อมูล
