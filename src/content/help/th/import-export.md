# นำเข้าและส่งออก

นำบันทึกของคุณเข้าสู่ Classic จากแอปอื่น หรือส่งออกเพื่อแชร์หรือสำรองข้อมูล Classic รองรับหลายรูปแบบสำหรับการย้ายข้อมูลที่ราบรื่น

## การนำเข้าบันทึก

### รูปแบบที่รองรับ

| รูปแบบ | นามสกุล | หมายเหตุ |
| ------ | --------- | ----- |
| Markdown | `.md` | ✅ รองรับเต็มรูปแบบ |
| Plain Text | `.txt` | ✅ นำเข้าเป็นบันทึก |
| JSON | `.json` | ✅ รูปแบบ Classic |
| HTML | `.html` | ⚠️ แปลงเป็น Markdown |
| Evernote | `.enex` | ⚠️ ทดลอง |
| Notion | การส่งออก Notion | ⚠️ ทดลอง |
| Obsidian | การส่งออก Markdown | ✅ รองรับเต็มรูปแบบ |

### กระบวนการนำเข้า

1. เปิด **File** → **Import** (หรือ `Cmd/Ctrl+I`)
2. เลือกไฟล์ของคุณที่จะนำเข้า
3. เลือกโฟลเดอร์ปลายทาง
4. ตรวจสอบตัวอย่างการนำเข้า
5. ยืนยันการนำเข้า

### นำเข้าจากคลิปบอร์ด

นำเข้าด่วนจากแหล่งใดก็ได้

1. คัดลอกข้อความไปยังคลิปบอร์ด
2. เปิด Command Palette
3. พิมพ์ "Import from Clipboard"
4. บันทึกใหม่ถูกสร้างขึ้นอัตโนมัติ

### การนำเข้าแบบกลุ่ม

นำเข้าหลายไฟล์พร้อมกัน

1. เปิด **File** → **Import**
2. เลือก **Import Folder**
3. เลือกโฟลเดอร์ที่มีบันทึก
4. ไฟล์ที่รองรับทั้งหมดถูกนำเข้า

## การส่งออกบันทึก

### รูปแบบการส่งออก

| รูปแบบ | กรณีการใช้งาน |
| ------ | ------- |
| **Markdown** | การแชร์, การสำรองข้อมูล, แอปอื่น |
| **HTML** | การเผยแพร่บนเว็บ |
| **PDF** | การเก็บถาวร, การพิมพ์ |
| **Plain Text** | การแชร์แบบง่าย |
| **JSON** | การรวม API |

### ส่งออกบันทึกเดียว

1. เปิดบันทึกที่คุณต้องการส่งออก
2. คลิก **File** → **Export**
3. เลือกรูปแบบ
4. เลือกตำแหน่ง
5. บันทึก

### ส่งออกหลายบันทึก

1. เลือกบันทึกในแถบด้านข้าง (Cmd/Ctrl+คลิก)
2. คลิกขวา → **Export Selected**
3. เลือกรูปแบบ
4. เลือกตำแหน่ง
5. บันทึก

### ส่งออกบันทึกทั้งหมด

1. เปิด **File** → **Export All**
2. เลือกรูปแบบ
3. เลือกตำแหน่ง
4. รอให้การส่งออกเสร็จสมบูรณ์

## คู่มือการย้ายข้อมูล

### จาก Obsidian

Classic สามารถนำเข้า Obsidian vaults โดยตรง

1. เปิด **File** → **Import**
2. เลือก **Obsidian Vault**
3. นำทางไปยังโฟลเดอร์ vault ของคุณ
4. เลือกเพื่อนำเข้า

**สิ่งที่ถูกถ่ายโอน**

- ✅ บันทึกและโฟลเดอร์
- ✅ แท็ก
- ✅ ลิงก์ (แปลงเป็นแบ็คลิงก์)
- ✅ เมทาดาทา
- ⚠️ ปลั๊กอิน (ไม่รองรับ)

### จาก Notion

1. ส่งออกจาก Notion เป็น Markdown
2. นำเข้าสู่ Classic
3. ตรวจสอบการจัดรูปแบบ
4. แก้ไขลิงก์ที่เสีย

**เคล็ดลับสำหรับการย้ายจาก Notion**

- ใช้ "Export as Markdown" ของ Notion
- ตรวจสอบการจัดรูปแบบ checkbox
- ตรวจสอบการอ้างอิงฐานข้อมูล
- สร้างฐานข้อมูลที่เชื่อมโยงใหม่ด้วยตนเอง

### จาก Evernote

1. ส่งออกจาก Evernote เป็น HTML
2. ใช้การนำเข้า HTML ของ Classic
3. ตรวจสอบเนื้อหาที่แปลงแล้ว

**เคล็ดลับสำหรับการย้ายจาก Evernote**

- Notebooks กลายเป็นโฟลเดอร์
- แท็กถูกรักษาไว้
- ไฟล์แนบถูกนำเข้า
- ตรวจสอบการจัดรูปแบบ

## แนวทางปฏิบัติที่ดีที่สุด

### ก่อนการนำเข้า

- สำรองข้อมูลบันทึกที่มีอยู่
- ตรวจสอบตัวอย่างการนำเข้า
- ตรวจสอบรายการที่ซ้ำกัน
- ตรวจสอบการจัดรูปแบบ

### หลังการนำเข้า

- ตรวจสอบบันทึกที่นำเข้า
- แก้ไขปัญหาการจัดรูปแบบ
- อัปเดตลิงก์ภายใน
- เพิ่มแท็กตามต้องการ

### การส่งออกปกติ

- ส่งออกการสำรองข้อมูลรายเดือน
- ใช้รูปแบบต่างกันสำหรับวัตถุประสงค์ต่างกัน
- เก็บการส่งออกจัดระเบียบตามวันที่
- เก็บในคลาวด์สำหรับการสำรองข้อมูลนอกสถานที่

## การแก้ไขปัญหา

### การนำเข้าล้มเหลว

**สาเหตุทั่วไป**

- รูปแบบไม่รองรับ
- ไฟล์เสียหาย
- ปัญหาสิทธิ์
- ข้อจำกัดหน่วยความจำ

**วิธีแก้ไข**

1. ลองรูปแบบอื่น
2. ตรวจสอบความสมบูรณ์ของไฟล์
3. ตรวจสอบสิทธิ์การอ่าน
4. นำเข้าไฟล์น้อยลงในแต่ละครั้ง

### การจัดรูปแบบสูญหาย

**สาเหตุทั่วไป**

- การจัดรูปแบบที่ซับซ้อน
- ไวยากรณ์ไม่รองรับ
- ข้อจำกัดการแปลง

**วิธีแก้ไข**

1. ใช้การส่งออก Markdown จากแอปต้นทาง
2. ทำให้การจัดรูปแบบง่ายขึ้นก่อนนำเข้า
3. แก้ไขด้วยตนเองหลังนำเข้า

### ไฟล์แนบหายไป

**สาเหตุทั่วไป**

- การนำเข้าไม่รวมไฟล์แนบ
- เส้นทางไฟล์แนบผิด
- ข้อจำกัดขนาด

**วิธีแก้ไข**

1. นำเข้าใหม่พร้อมไฟล์แนบ
2. ตรวจสอบโฟลเดอร์ไฟล์แนบ
3. นำเข้าไฟล์แนบแยกต่างหาก
