# ประวัติเวอร์ชัน

Classic ติดตามการเปลี่ยนแปลงทุกอย่างในบันทึกของคุณ เรียกดู เปรียบเทียบ และกู้คืนเวอร์ชันก่อนหน้าเมื่อใดก็ตามที่คุณต้องการ

## ประวัติเวอร์ชันทำงานอย่างไร

### การติดตามอัตโนมัติ

Classic บันทึกเวอร์ชันของบันทึกของคุณโดยอัตโนมัติ

- **ทุกการแก้ไข**: แต่ละครั้งที่บันทึกจะสร้างเวอร์ชัน
- **ประทับเวลา**: รู้ว่าการเปลี่ยนแปลงเกิดขึ้นเมื่อไหร่
- **เนื้อหาเต็ม**: สแนปช็อตที่สมบูรณ์ถูกเก็บรักษาไว้

### การเก็บรักษา

| การตั้งค่า | ค่าเริ่มต้น |
|---------|---------|
| **การเก็บรักษาประวัติ** | 90 วัน |
| **เวอร์ชันสูงสุด** | 100 ต่อบันทึก |
| **การทำความสะอาดอัตโนมัติ** | ใช่ |

กำหนดค่าเหล่านี้ใน Settings → Editor → Version History

## การดูประวัติ

### การเปิดประวัติเวอร์ชัน

1. เปิดบันทึกใดๆ
2. คลิกไอคอนประวัติในแถบเครื่องมือ
3. หรือใช้ `Cmd/Ctrl+H`

### แผงประวัติ

แผงประวัติแสดง

- **ไทม์ไลน์**: รายการเวอร์ชันทั้งหมด
- **ประทับเวลา**: เมื่อแต่ละเวอร์ชันถูกบันทึก
- **ตัวอย่าง**: ดูเนื้อหาของเวอร์ชันที่เลือก

### การนำทางเวอร์ชัน

1. คลิกเวอร์ชันใดๆ ในรายการ
2. ตัวอย่างแสดงเวอร์ชันนั้น
3. เลื่อนดูการเปลี่ยนแปลง
4. เปรียบเทียบกับเวอร์ชันปัจจุบัน

## การเปรียบเทียบเวอร์ชัน

### มุมมองเคียงข้างกัน

ดูความแตกต่างระหว่างเวอร์ชัน

1. เปิดประวัติเวอร์ชัน
2. เลือกเวอร์ชัน
3. คลิก "Compare"
4. ดูส่วนที่เพิ่ม (เขียว) และลบ (แดง)

### มุมมอง Diff

ความแตกต่างของข้อความถูกเน้น

- **เขียว**: เนื้อหาที่เพิ่ม
- **แดง**: เนื้อหาที่ลบ
- **ไม่เปลี่ยนแปลง**: ข้อความปกติ

## การกู้คืนเวอร์ชัน

### กู้คืนบันทึกทั้งหมด

แทนที่เนื้อหาปัจจุบันด้วยเวอร์ชันก่อนหน้า

1. เปิดประวัติเวอร์ชัน
2. เลือกเวอร์ชัน
3. คลิก "Restore"
4. ยืนยันการดำเนินการ

### กู้คืนส่วนที่เลือก

คัดลอกเฉพาะส่วนของเวอร์ชันก่อนหน้า

1. เปิดประวัติเวอร์ชัน
2. เลือกเวอร์ชัน
3. เน้นข้อความที่คุณต้องการ
4. คลิก "Copy to Clipboard"
5. วางในบันทึกปัจจุบันของคุณ

### สร้างบันทึกใหม่จากเวอร์ชัน

เก็บทั้งสองเวอร์ชัน

1. เปิดประวัติเวอร์ชัน
2. เลือกเวอร์ชัน
3. คลิก "Create Note from Version"
4. บันทึกใหม่ถูกสร้างขึ้น

## การดำเนินการประวัติเวอร์ชัน

| การดำเนินการ | คำอธิบาย |
|--------|-------------|
| **ดู** | ดูตัวอย่างเวอร์ชัน |
| **เปรียบเทียบ** | Diff กับปัจจุบัน |
| **กู้คืน** | แทนที่ปัจจุบันด้วยอันนี้ |
| **คัดลอก** | คัดลอกเนื้อหาไปยังคลิปบอร์ด |
| **ลบ** | ลบเวอร์ชันนี้ |

## ทางลัดคีย์บอร์ด

| การดำเนินการ | ทางลัด |
|--------|----------|
| เปิดประวัติ | `Cmd/Ctrl+H` |
| เวอร์ชันก่อนหน้า | `Up` |
| เวอร์ชันถัดไป | `Down` |
| กู้คืน | `Cmd/Ctrl+R` |
| ปิดประวัติ | `Escape` |

## การตั้งค่า

### กำหนดค่าใน Settings → Editor → Version History

| การตั้งค่า | ตัวเลือก | ค่าเริ่มต้น |
|---------|---------|---------|
| เปิดใช้งานประวัติ | เปิด/ปิด | เปิด |
| ระยะเวลาเก็บรักษา | 30/60/90/180 วัน | 90 วัน |
| เวอร์ชันสูงสุด | 50/100/200/500 | 100 |
| ช่วง Auto-save | 1/5/10/30 นาที | 5 นาที |

### ตำแหน่งจัดเก็บ

ประวัติเวอร์ชันถูกเก็บไว้ในเครื่อง

- **macOS**: `~/Library/Application Support/Classic/history/`
- **Windows**: `%APPDATA%/Classic/history/`
- **Linux**: `~/.config/Classic/history/`

## กรณีการใช้งาน

### การกู้คืนการเปลี่ยนแปลงโดยไม่ตั้งใจ

1. เปิดประวัติเวอร์ชัน
2. ค้นหาเวอร์ชันก่อนความผิดพลาด
3. กู้คืนเวอร์ชันนั้น

### การเปรียบเทียบฉบับร่าง

1. เปิดประวัติ
2. เปรียบเทียบเวอร์ชันต่างๆ
3. ดูว่าบันทึกพัฒนาอย่างไร

### การค้นหาเนื้อหาที่ลบ

1. เรียกดูเวอร์ชันเก่า
2. ค้นหาข้อความที่ลบ
3. คัดลอกกลับมา

### การติดตามความคืบหน้า

ดูว่าบันทึกพัฒนาตลอดเวลา

- เมื่อแนวคิดถูกเพิ่ม
- โครงสร้างเปลี่ยนแปลงอย่างไร
- วิวัฒนาการของการเขียน

## เคล็ดลับ

### การตรวจสอบปกติ

ตรวจสอบประวัติเวอร์ชันเป็นระยะ

- ทำความสะอาดเวอร์ชันที่ไม่จำเป็น
- บันทึกว่าเนื้อหาพัฒนาอย่างไร
- เรียนรู้จากรูปแบบการแก้ไขของคุณ

### ก่อนการเปลี่ยนแปลงใหญ่

สร้างจุดตรวจสอบ

1. ทำการแก้ไขเล็กน้อย
2. บันทึก (สร้างเวอร์ชัน)
3. ตอนนี้ทำการเปลี่ยนแปลงใหญ่ของคุณ
4. ง่ายต่อการย้อนกลับหากจำเป็น

### ตั้งชื่อเวอร์ชันสำคัญ

เพิ่มตัวระบุในบันทึกของคุณ

```markdown
<!-- VERSION: Final draft -->
<!-- VERSION: Before restructuring -->
```

เหล่านี้ช่วยระบุเวอร์ชันในภายหลัง

## การแก้ไขปัญหา

### ประวัติไม่บันทึก

- ตรวจสอบว่าประวัติเวอร์ชันเปิดใช้งานหรือไม่
- ตรวจสอบว่าตำแหน่งจัดเก็บมีพื้นที่
- ตรวจสอบสิทธิ์ไฟล์

### ไม่พบเวอร์ชันเก่า

- เวอร์ชันอาจถูกทำความสะอาดอัตโนมัติ
- ตรวจสอบการตั้งค่าการเก็บรักษา
- เพิ่มระยะเวลาการเก็บรักษา

### ประวัติใช้พื้นที่มากเกินไป

1. ลดระยะเวลาการเก็บรักษา
2. ลดเวอร์ชันสูงสุดต่อบันทึก
3. รันการทำความสะอาดด้วยตนเอง

### การโหลดประวัติช้า

บันทึกขนาดใหญ่ที่มีหลายเวอร์ชันอาจโหลดช้า

- อดทนกับประวัติขนาดใหญ่
- พิจารณาลดเวอร์ชันสูงสุด
- เก็บถาวรบันทึกเก่า

## ความเป็นส่วนตัวและความปลอดภัย

### การจัดเก็บในเครื่อง

ประวัติเวอร์ชันถูกเก็บไว้ในเครื่อง

- ไม่เคยส่งไปยังเซิร์ฟเวอร์
- ข้อมูลของคุณเป็นส่วนตัว
- มีเพียงคุณที่เข้าถึงได้

### การสำรองข้อมูลแบบเข้ารหัส

หากคุณใช้การสำรองข้อมูลแบบเข้ารหัส

- ประวัติเวอร์ชันรวมอยู่ด้วย
- ป้องกันด้วยรหัสผ่านสำรองข้อมูลของคุณ
- ปลอดภัยแม้ในไฟล์สำรองข้อมูล
