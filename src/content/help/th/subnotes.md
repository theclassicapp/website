# Subnotes

Subnotes ช่วยให้คุณสร้างบันทึกที่ซ้อนกันและเป็นลำดับชั้นภายใน Classic จัดระเบียบหัวข้อที่ซับซ้อนเป็นส่วนๆ ที่จัดการได้

## Subnotes คืออะไร?

Subnotes เป็นบันทึกภายในบันทึก—วิธีสร้างโครงสร้างเนื้อหาแบบลำดับชั้น

### ประโยชน์

- **การจัดระเบียบ**: แบ่งหัวข้อที่ซับซ้อนเป็นส่วนเล็กๆ
- **การซ้อน**: สร้างหลายระดับของลำดับชั้น
- **การนำทาง**: นำทางระหว่างบันทึกที่เกี่ยวข้องได้ง่าย
- **ความยืดหยุ่น**: ขยายและยุบตามต้องการ

## การสร้าง Subnotes

### วิธีที่ 1: การเยื้อง

สร้าง subnotes โดยการเยื้องเนื้อหาภายใต้บันทึกหลัก

1. สร้างบันทึกหลัก
2. สร้างบันทึกใหม่
3. ลากบันทึกใหม่ไปวางบนบันทึกหลัก
4. มันจะกลายเป็น subnote

### วิธีที่ 2: เมนูบริบท

1. คลิกขวาที่บันทึก
2. เลือก "New Subnote"
3. บันทึกที่ซ้อนจะถูกสร้างขึ้น

### วิธีที่ 3: ทางลัดคีย์บอร์ด

1. เลือกบันทึก
2. กด `Cmd/Ctrl+Shift+N`
3. Subnote จะถูกสร้างขึ้น

## การจัดการ Subnotes

### การขยายและยุบ

คลิกลูกศรข้างบันทึกที่มี subnotes

- **ขยาย**: แสดงบันทึกที่ซ้อนทั้งหมด
- **ยุบ**: ซ่อนบันทึกที่ซ้อน

### การนำทาง

- คลิกเพื่อเปิด subnote
- ใช้ปุ่มลูกศรเพื่อนำทางลำดับชั้น
- Breadcrumb แสดงตำแหน่งของคุณ

### การย้าย Subnotes

ลากและวางเพื่อจัดระเบียบใหม่

1. คลิกและกดที่ subnote
2. ลากไปยังตำแหน่งใหม่
3. วางเพื่อย้ายตำแหน่ง

### การแปลงเป็นบันทึกหลัก

เลื่อนระดับ subnote เป็นระดับบนสุด

1. คลิกขวาที่ subnote
2. เลือก "Move to Root"
3. บันทึกจะกลายเป็นบันทึกหลัก

## ความลึกของ Subnote

Classic รองรับหลายระดับของการซ้อน

```
📁 Project
  📄 Overview
  📁 Research
    📄 User Interviews
    📄 Survey Results
    📁 Analysis
      📄 Key Findings
      📄 Recommendations
  📁 Planning
    📄 Timeline
    📄 Resources
```

### แนวทางปฏิบัติที่ดีที่สุด

- **จำกัดความลึก**: สูงสุด 3-4 ระดับเพื่อความชัดเจน
- **การจัดกลุ่มตามตรรกะ**: จัดกลุ่มเนื้อหาที่เกี่ยวข้องเข้าด้วยกัน
- **การตั้งชื่อที่ชัดเจน**: ใช้ชื่อที่สื่อความหมายสำหรับ subnotes

## การเชื่อมโยงกับ Subnotes

### แบ็คลิงก์

Subnotes สร้างแบ็คลิงก์ไปยังบันทึกหลักโดยอัตโนมัติ

- ดูส่วน "Linked References"
- นำทางระหว่างบันทึกที่เกี่ยวข้อง
- เข้าใจความสัมพันธ์ของบันทึก

### Wikilinks

เชื่อมโยงไปยัง subnotes จากที่ไหนก็ได้

```markdown
See [[Project/Research/Analysis]] for details.
```

## เคล็ดลับ

### หลักการตั้งชื่อ

ใช้การตั้งชื่อที่ชัดเจนและสอดคล้อง

```
✅ ดี:
Project/Requirements/Functional
Project/Requirements/Non-Functional

❌ หลีกเลี่ยง:
Project/req/func
Project/rq/nf
```

### การนำทาง Breadcrumb

ใช้ breadcrumbs เพื่อเข้าใจตำแหน่งของคุณ

```
Project > Research > Analysis
```

### การค้นหา

การค้นหารวม subnotes โดยอัตโนมัติ

- ค้นหาเนื้อหาในทุกระดับ
- กรองตามความลึกของบันทึก
- การนำทางด่วน

### เทมเพลต

ใช้เทมเพลตกับ subnotes

1. สร้าง subnote
2. ใช้ slash command `/template`
3. เลือกเทมเพลต

## ทางลัดคีย์บอร์ด

| การดำเนินการ | ทางลัด |
|--------|----------|
| สร้าง Subnote ใหม่ | `Cmd/Ctrl+Shift+N` |
| ขยายทั้งหมด | `Cmd/Ctrl+Down` |
| ยุบทั้งหมด | `Cmd/Ctrl+Up` |
| เลื่อนขึ้น | `Opt/Alt+Up` |
| เลื่อนลง | `Opt/Alt+Down` |
| เลื่อนระดับขึ้น | `Opt/Alt+Left` |
| เลื่อนระดับลง | `Opt/Alt+Right` |

## การแก้ไขปัญหา

### Subnote ไม่แสดง

- ตรวจสอบว่าบันทึกหลักถูกยุบหรือไม่
- ตรวจสอบว่าบันทึกไม่ถูกย้าย
- รีเฟรชแถบด้านข้าง

### ไม่สามารถสร้าง Subnote

- ตรวจสอบให้แน่ใจว่าบันทึกหลักมีอยู่
- ตรวจสอบสิทธิ์
- ลองใช้วิธีเมนูบริบท

### สูญเสีย Subnotes

ใช้การค้นหาเพื่อหาบันทึกที่หายไป

1. เปิดการค้นหา (`Cmd/Ctrl+F`)
2. พิมพ์ชื่อบันทึก
3. ดูเส้นทางเต็มในผลลัพธ์
