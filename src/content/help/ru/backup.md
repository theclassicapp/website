# Резервное копирование и восстановление

Держите заметки в безопасности со встроенной системой резервного копирования Classic. Никогда больше не теряйте работу.

## Автоматическое резервное копирование

### Как это работает

Classic автоматически создаёт резервные копии заметок через регулярные интервалы:

- **Частота по умолчанию**: Каждые 30 минут
- **Расположение резервных копий**: Та же папка, что и заметки
- **Хранение**: Хранит последние 10 резервных копий
- **Формат**: Сжатый архив (`.cbak`)

### Настройка автосохранения

1. Откройте настройки
2. Перейдите в **Резервное копирование и восстановление**
3. Настройте следующее:

| Параметр | Опции | По умолчанию |
| ------- | ------- | ------- |
| Включить автосохранение | Вкл/Выкл | Вкл |
| Частота резервного копирования | 15мин / 30мин / 1ч / 4ч | 30мин |
| Хранить резервные копии | 5 / 10 / 20 / 50 | 10 |
| Расположение резервных копий | Пользовательский путь | Папка заметок |

## Ручное резервное копирование

### Создать резервную копию сейчас

1. Откройте палитру команд (`Cmd/Ctrl+P`)
2. Введите "Резервная копия"
3. Выберите **Создать резервную копию сейчас**
4. Выберите расположение (опционально)
5. Резервная копия создана

### Формат файла резервной копии

Резервные копии хранятся как файлы `.cbak`, содержащие:

- Все заметки (файлы Markdown)
- Вложения
- Настройки
- Метаданные (теги, ссылки и т.д.)

## Восстановление из резервной копии

### Процесс восстановления

1. Откройте настройки
2. Перейдите в **Резервное копирование и восстановление**
3. Нажмите **Восстановить из резервной копии**
4. Выберите файл резервной копии
5. Выберите опции восстановления
6. Подтвердите восстановление

### Опции восстановления

| Опция | Описание |
| ------ | ----------- |
| **Заменить всё** | Полная замена текущих заметок резервной копией |
| **Объединить** | Объединение резервной копии с существующими заметками |
| **Предпросмотр** | Просмотр содержимого резервной копии перед восстановлением |

### Что восстанавливается

- ✅ Все заметки и папки
- ✅ Вложения
- ✅ Теги и метаданные
- ✅ История заметок
- ⚠️ Настройки (опционально)
- ❅ Настройки приложения

## Расположения резервных копий

### Расположение по умолчанию

```
~/Classic/backups/
├── backup-2024-01-15-10-30.cbak
├── backup-2024-01-15-11-00.cbak
└── backup-2024-01-15-11-30.cbak
```

### Пользовательское расположение

1. Откройте настройки
2. Перейдите в Резервное копирование и восстановление
3. Нажмите "Изменить расположение резервных копий"
4. Выберите предпочтительную папку

### Облачное резервное копирование

Укажите расположение резервных копий в папке с облачной синхронизацией:

- **iCloud**: `~/Library/Mobile Documents/com~apple~CloudDocs/Classic Backups`
- **Dropbox**: `~/Dropbox/Classic Backups`
- **Google Drive**: `~/Google Drive/Classic Backups`
- **OneDrive**: `~/OneDrive/Classic Backups`

## Лучшие практики

### Расписание резервного копирования

- **Автосохранение**: Держите включённым с 30-минутными интервалами
- **Ручное резервное копирование**: Создавайте перед значительными изменениями
- **Внешнее резервное копирование**: Регулярно копируйте резервные копии на внешнее хранилище

### Верификация резервных копий

Периодически проверяйте резервные копии:

1. Создайте тестовое восстановление
2. Проверьте содержимое заметок
3. Верифицируйте вложения
4. Подтвердите метаданные

### Ротация резервных копий

- Храните недавние резервные копии локально
- Архивируйте старые резервные копии на внешнее хранилище
- Удаляйте очень старые резервные копии для экономии места

## Устранение неполадок

### Ошибка резервного копирования

**Возможные причины**

- Недостаточно места на диске
- Проблемы с правами
- Файл используется

**Решения**

1. Проверьте доступное место на диске
2. Верифицируйте права папки
3. Закройте другие приложения
4. Попробуйте другое расположение

### Ошибка восстановления

**Возможные причины**

- Повреждённый файл резервной копии
- Несовместимая версия
- Недостаточно места

**Решения**

1. Попробуйте более старую резервную копию
2. Проверьте целостность файла резервной копии
3. Освободите место на диске
4. Обратитесь в поддержку

### Отсутствующие резервные копии

**Возможные причины**

- Автосохранение отключено
- Расположение резервных копий изменено
- Резервные копии удалены

**Решения**

1. Проверьте настройки резервного копирования
2. Верифицируйте расположение резервных копий
3. Включите автосохранение
4. Создайте резервную копию вручную сейчас

## Расширенные опции

### Шифрование резервных копий

Для дополнительной безопасности:

1. Откройте настройки
2. Перейдите в Резервное копирование и восстановление
3. Включите "Шифровать резервные копии"
4. Установите пароль
5. Запомните этот пароль — он не может быть восстановлен

### Инкрементальные резервные копии

Экономьте место с инкрементальными резервными копиями:

- Хранятся только изменения с последней резервной копии
- Полное восстановление реконструирует полное состояние
- Уменьшает время и размер резервного копирования

### Исключения из резервного копирования

Исключите определённый контент из резервных копий:

1. Откройте настройки
2. Перейдите в Резервное копирование и восстановление
3. Нажмите "Исключения"
4. Добавьте папки или файлы для исключения
