# ייבוא וייצוא

הביאו את ההערות שלכם ל-Classic מאפליקציות אחרות, או יצאו אותן לשיתוף או גיבוי. Classic תומך בפורמטים מרובים להעברת נתונים חלקה.

## ייבוא הערות

### פורמטים נתמכים

| פורמט | סיומת | הערות |
| ------ | --------- | ----- |
| Markdown | `.md` | ✅ תמיכה מלאה |
| טקסט פשוט | `.txt` | ✅ מיובא כהערה |
| JSON | `.json` | ✅ פורמט Classic |
| HTML | `.html` | ⚠️ מומר ל-Markdown |
| Evernote | `.enex` | ⚠️ ניסיוני |
| Notion | ייצוא Notion | ⚠️ ניסיוני |
| Obsidian | ייצוא Markdown | ✅ תמיכה מלאה |

### תהליך ייבוא

1. פתחו **קובץ** → **ייבוא** (או `Cmd/Ctrl+I`)
2. בחרו את הקובץ/ים לייבוא
3. בחרו תיקיית יעד
4. סקרו תצוגה מקדימה של ייבוא
5. אשרו ייבוא

### ייבוא מלוח

ייבוא מהיר מכל מקור

1. העתיקו טקסט ללוח
2. פתחו פלטת פקודות
3. הקלידו "ייבוא מלוח"
4. נוצרת הערה חדשה אוטומטית

### ייבוא כמותי

ייבאו קבצים מרובים בבת אחת

1. פתחו **קובץ** → **ייבוא**
2. בחרו **ייבוא תיקייה**
3. בחרו את התיקייה המכילה הערות
4. כל הקבצים הנתמכים מיובאים

## ייצוא הערות

### פורמטים ייצוא

| פורמט | מקרה שימוש |
| ------ | ------- |
| **Markdown** | שיתוף, גיבויים, אפליקציות אחרות |
| **HTML** | פרסום באינטרנט |
| **PDF** | ארכוב, הדפסה |
| **טקסט פשוט** | שיתוף פשוט |
| **JSON** | אינטגרציית API |

### ייצוא הערה בודדת

1. פתחו את ההערה
2. לחצו **קובץ** → **ייצוא**
3. בחרו פורמט
4. בחרו מיקום
5. שמרו

### ייצוא הערות מרובות

1. בחרו הערות בסרגל הצד (Cmd/Ctrl+לחיצה)
2. לחיצה ימנית → **ייצוא נבחרות**
3. בחרו פורמט
4. בחרו מיקום
5. שמרו

### ייצוא כל ההערות

1. פתחו **קובץ** → **ייצוא הכל**
2. בחרו פורמט
3. בחרו מיקום
4. המתינו לסיום הייצוא

## מדריכי העברה

### מ-Obsidian

Classic יכול לייבא כספות Obsidian ישירות

1. פתחו **קובץ** → **ייבוא**
2. בחרו **כספת Obsidian**
3. נווטו לתיקיית הכספת שלכם
4. בחרו לייבא

**מה עובר**

- ✅ הערות ותיקיות
- ✅ תגיות
- ✅ קישורים (מומרים לקישורים חוזרים)
- ✅ מטא-נתונים
- ⚠️ תוספים (לא נתמכים)

### מ-Notion

1. יצאו מ-Notion כ-Markdown
2. ייבאו ל-Classic
3. סקרו עיצוב
4. תקנו קישורים שבורים

**טיפים להעברה מ-Notion**

- השתמשו ב"ייצא כ-Markdown" של Notion
- בדקו עיצוב תיבות סימון
- ודאו הפניות למסדי נתונים
- צרו מחדש מסדי נתונים מקושרים ידנית

### מ-Evernote

1. יצאו מ-Evernote כ-HTML
2. השתמשו בייבוא HTML של Classic
3. סקרו תוכן שהומר

**טיפים להעברה מ-Evernote**

- מחברות הופכות לתיקיות
- תגיות נשמרות
- קבצים מצורפים מיובאים
- בדקו עיצוב

## שיטות עבודה מומלצות

### לפני ייבוא

- גבו הערות קיימות
- סקרו תצוגה מקדימה של ייבוא
- בדקו כפילויות
- ודאו עיצוב

### אחרי ייבוא

- סקרו הערות שיובאו
- תקנו בעיות עיצוב
- עדכנו קישורים פנימיים
- הוסיפו תגיות לפי הצורך

### ייצוא קבוע

- יצאו גיבויים חודשית
- השתמשו בפורמטים שונים למטרות שונות
- ארגנו ייצוא לפי תאריך
- אחסנו בענן לגיבוי מחוץ לאתר

## פתרון בעיות

### ייבוא נכשל

**גורמים נפוצים**

- פורמט לא נתמך
- קובץ פגום
- בעיות הרשאות
- מגבלות זיכרון

**פתרונות**

1. נסו פורמט אחר
2. בדקו שלמות קובץ
3. ודאו הרשאות קריאה
4. ייבאו פחות קבצים בכל פעם

### עיצוב אבד

**גורמים נפוצים**

- עיצוב מורכב
- תחביר לא נתמך
- מגבלות המרה

**פתרונות**

1. השתמשו בייצוא Markdown מאפליקציית המקור
2. פשטו עיצוב לפני ייבוא
3. תקנו ידנית לאחר ייבוא

### קבצים מצורפים חסרים

**גורמים נפוצים**

- ייבוא לא כלל קבצים מצורפים
- נתיב קובץ מצורף שגוי
- מגבלות גודל

**פתרונות**

1. ייבאו מחדש עם קבצים מצורפים
2. בדקו תיקיית קבצים מצורפים
3. ייבאו קבצים מצורפים בנפרד
