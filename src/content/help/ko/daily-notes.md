# 데일리 노트

데일리 노트는 매일 자동으로 새 노트를 만들어 저널링, 일일 계획 또는 시간 기반 정보 캡처에 완벽합니다.

## 작동 방식

### 자동 생성

- 매일 새 노트가 생성됩니다
- 노트는 날짜 형식을 사용하여 이름이 지정됩니다 (예: `2024-01-15.md`)
- 이전 날짜의 노트는 보존되고 검색 가능합니다

### 데일리 노트 접근

1. 사이드바에서 **데일리 노트** 클릭
2. 또는 `Ctrl/Cmd+D` 누르기
3. 오늘의 노트가 자동으로 열립니다

## 데일리 노트 템플릿

새 데일리 노트에 대한 템플릿 사용자 정의

### 기본 템플릿

```markdown
# {{date}}

## 작업
- [ ]
- [ ]

## 노트


## 저널

```

### 사용자 정의

1. 설정 열기
2. 데일리 노트로 이동
3. 템플릿 편집
4. 현재 날짜에 `{{date}}` 사용

### 템플릿 변수

| 변수 | 설명 |
| ---- | ---- |
| `{{date}}` | 현재 날짜 (YYYY-MM-DD) |
| `{{time}}` | 현재 시간 (HH:mm) |
| `{{weekday}}` | 요일 이름 (월요일 등) |

## 기능

### 날짜 탐색

- **이전/다음**: 날짜 간 탐색
- **달력**: 특정 날짜로 이동
- **검색**: 모든 데일리 노트에서 콘텐츠 찾기

### 자동 연결

- 같은 날 생성된 노트는 자동으로 연결됩니다
- 백링크는 날짜 간 연결을 표시합니다
- 태그는 모든 데일리 노트에서 작동합니다

### 통합

- **빠른 캡처**: `#today`로 오늘의 노트에 추가
- **달력 보기**: 데일리 노트의 시각적 개요
- **통계**: 글쓰기 연속 기록 추적

## 모범 사례

### 일일 저널링

```markdown
# 2024-01-15

## 오전
- 6:30에 기상
- 기분: 활기참

## 작업
- [x] 오전 운동
- [ ] PR 리뷰
- [ ] 오후 2시 팀 회의

## 배운 것
- 새로운 키보드 단축키 발견
- Mermaid 다이어그램에 대해 배움

## 내일
- 문서화에 집중
- 코드 리뷰 예약
```

### 회의 노트

```markdown
# 2024-01-15

## 팀 스탠드업

**참석자**: Alice, Bob, Carol

### 업데이트
- Alice: API 통합 완료
- Bob: UI 컴포넌트 작업 중
- Carol: 테스트 진행 중

### 차단 사항
- 디자인 리뷰 대기 중

### 실행 항목
- [ ] Alice: 스테이징에 배포
- [ ] Bob: 컴포넌트 라이브러리 완성
- [ ] Carol: 통합 테스트 작성
```

### 학습 로그

```markdown
# 2024-01-15

## 오늘 배운 것

### Mermaid 다이어그램
- Markdown에서 순서도를 만들 수 있음
- 구문이 간단함
- 실시간 미리보기가 도움이 됨

### 리소스
- [Mermaid 문서](https://mermaid.js.org/)

## 질문
- 복잡한 다이어그램을 만드는 방법?
- 다이어그램 정리 모범 사례?
```

## 팁

### 일관성

- 매일 같은 시간에 노트 만들기
- 일관된 서식 사용
- 하루 종일 검토 및 업데이트

### 정리

- 다른 섹션에 제목 사용
- 쉬운 검색을 위해 태그 추가
- 관련 노트에 연결

### 검토

- 데일리 노트 주간 검토
- 중요한 항목을 영구 노트로 추출
- 오래된 노트 보관 또는 요약
