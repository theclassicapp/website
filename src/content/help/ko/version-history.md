# 버전 기록

Classic은 노트의 모든 변경 사항을 추적합니다. 필요할 때마다 이전 버전을 탐색, 비교 및 복원하세요.

## 버전 기록 작동 방식

### 자동 추적

Classic은 자동으로 노트의 버전을 저장합니다

- **모든 편집**: 각 저장이 버전을 만듭니다
- **타임스탬프**: 변경이 발생한 시간을 알 수 있습니다
- **전체 콘텐츠**: 완전한 스냅샷이 보존됩니다

### 보존

| 설정 | 기본값 |
|------|--------|
| **기록 보존** | 90일 |
| **최대 버전** | 노트당 100개 |
| **자동 정리** | 예 |

설정 → 에디터 → 버전 기록에서 구성하세요.

## 기록 보기

### 버전 기록 열기

1. 노트 열기
2. 도구 모음의 기록 아이콘 클릭
3. 또는 `Cmd/Ctrl+H` 사용

### 기록 패널

기록 패널은 다음을 표시합니다

- **타임라인**: 모든 버전 목록
- **타임스탬프**: 각 버전이 저장된 시간
- **미리보기**: 선택한 버전의 콘텐츠 보기

### 버전 탐색

1. 목록에서 버전 클릭
2. 미리보기에 해당 버전이 표시됩니다
3. 변경 사항 스크롤
4. 현재 버전과 비교

## 버전 비교

### 나란히 보기

버전 간의 차이점 보기

1. 버전 기록 열기
2. 버전 선택
3. "비교" 클릭
4. 추가 (녹색) 및 삭제 (빨간색) 보기

### Diff 보기

텍스트 차이가 강조됩니다

- **녹색**: 추가된 콘텐츠
- **빨간색**: 제거된 콘텐츠
- **변경 없음**: 일반 텍스트

## 버전 복원

### 전체 노트 복원

현재 콘텐츠를 이전 버전으로 교체

1. 버전 기록 열기
2. 버전 선택
3. "복원" 클릭
4. 작업 확인

### 선택 복원

이전 버전의 일부만 복사

1. 버전 기록 열기
2. 버전 선택
3. 원하는 텍스트 강조
4. "클립보드에 복사" 클릭
5. 현재 노트에 붙여넣기

### 버전에서 새 노트 만들기

두 버전 모두 유지

1. 버전 기록 열기
2. 버전 선택
3. "버전에서 노트 만들기" 클릭
4. 새 노트가 생성됩니다

## 버전 기록 작업

| 작업 | 설명 |
|------|------|
| **보기** | 버전 미리보기 |
| **비교** | 현재와 Diff |
| **복원** | 현재를 이것으로 교체 |
| **복사** | 콘텐츠를 클립보드에 복사 |
| **삭제** | 이 버전 제거 |

## 키보드 단축키

| 작업 | 단축키 |
|------|--------|
| 기록 열기 | `Cmd/Ctrl+H` |
| 이전 버전 | `Up` |
| 다음 버전 | `Down` |
| 복원 | `Cmd/Ctrl+R` |
| 기록 닫기 | `Escape` |

## 설정

### 설정 → 에디터 → 버전 기록에서 구성

| 설정 | 옵션 | 기본값 |
|------|------|--------|
| 기록 활성화 | 켜기/끄기 | 켜기 |
| 보존 기간 | 30/60/90/180일 | 90일 |
| 최대 버전 | 50/100/200/500 | 100 |
| 자동 저장 간격 | 1/5/10/30분 | 5분 |

### 저장소 위치

버전 기록은 로컬에 저장됩니다

- **macOS**: `~/Library/Application Support/Classic/history/`
- **Windows**: `%APPDATA%/Classic/history/`
- **Linux**: `~/.config/Classic/history/`

## 사용 사례

### 실수로 변경 복구

1. 버전 기록 열기
2. 실수 전 버전 찾기
3. 해당 버전 복원

### 초안 비교

1. 기록 열기
2. 다른 버전 비교
3. 노트가 어떻게 발전했는지 보기

### 삭제된 콘텐츠 찾기

1. 오래된 버전 탐색
2. 삭제된 텍스트 검색
3. 다시 복사

### 진행 추적

시간이 지남에 따라 노트가 어떻게 발전했는지 보기

- 아이디어가 추가된 시간
- 구조가 어떻게 변경되었는지
- 글쓰기 발전

## 팁

### 정기 검토

정기적으로 버전 기록 검토

- 불필요한 버전 정리
- 콘텐츠가 어떻게 발전했는지 주목
- 편집 패턴에서 배우기

### 주요 변경 전

체크포인트 만들기

1. 작은 편집 만들기
2. 저장 (버전 생성)
3. 이제 주요 변경 만들기
4. 필요한 경우 쉽게 되돌리기

### 중요한 버전 이름 지정

노트에 마커 추가

```markdown
<!-- VERSION: 최종 초안 -->
<!-- VERSION: 재구성 전 -->
```

이것은 나중에 버전을 식별하는 데 도움이 됩니다.

## 문제 해결

### 기록이 저장되지 않음

- 버전 기록이 활성화되어 있는지 확인
- 저장소 위치에 공간이 있는지 확인
- 파일 권한 확인

### 오래된 버전을 찾을 수 없음

- 버전이 자동 정리되었을 수 있음
- 보존 설정 확인
- 보존 기간 늘리기

### 기록이 너무 많은 공간 차지

1. 보존 기간 줄이기
2. 노트당 최대 버전 낮추기
3. 수동 정리 실행

### 기록 로딩이 느림

많은 버전이 있는 큰 노트는 로딩이 느릴 수 있습니다

- 큰 기록은 인내심을 가지세요
- 최대 버전 줄이기 고려
- 오래된 노트 보관

## 개인정보 및 보안

### 로컬 저장소

버전 기록은 로컬에 저장됩니다

- 서버로 전송되지 않음
- 데이터가 비공개로 유지됨
- 사용자만 액세스 가능

### 암호화된 백업

암호화된 백업을 사용하는 경우

- 버전 기록이 포함됨
- 백업 암호로 보호됨
- 백업 파일에서도 안전함
