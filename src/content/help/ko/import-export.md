# 가져오기 및 내보내기

다른 앱에서 Classic으로 노트를 가져오거나 공유 또는 백업을 위해 내보내세요. Classic은 원활한 데이터 마이그레이션을 위해 여러 형식을 지원합니다.

## 노트 가져오기

### 지원되는 형식

| 형식 | 확장자 | 비고 |
| ---- | ------ | ---- |
| Markdown | `.md` | ✅ 전체 지원 |
| 일반 텍스트 | `.txt` | ✅ 노트로 가져오기 |
| JSON | `.json` | ✅ Classic 형식 |
| HTML | `.html` | ⚠️ Markdown으로 변환 |
| Evernote | `.enex` | ⚠️ 실험적 |
| Notion | Notion 내보내기 | ⚠️ 실험적 |
| Obsidian | Markdown 내보내기 | ✅ 전체 지원 |

### 가져오기 프로세스

1. **파일** → **가져오기** 열기 (또는 `Cmd/Ctrl+I`)
2. 가져올 파일 선택
3. 대상 폴더 선택
4. 가져오기 미리보기 검토
5. 가져오기 확인

### 클립보드에서 가져오기

모든 소스에서 빠른 가져오기

1. 클립보드에 텍스트 복사
2. 명령 팔레트 열기
3. "클립보드에서 가져오기" 입력
4. 새 노트가 자동으로 생성됩니다

### 대량 가져오기

여러 파일을 한 번에 가져오기

1. **파일** → **가져오기** 열기
2. **폴더 가져오기** 선택
3. 노트가 포함된 폴더 선택
4. 지원되는 모든 파일이 가져와집니다

## 노트 내보내기

### 내보내기 형식

| 형식 | 용도 |
| ---- | ---- |
| **Markdown** | 공유, 백업, 다른 앱 |
| **HTML** | 웹 게시, 보기 |
| **PDF** | 보관, 인쇄 |
| **일반 텍스트** | 간단한 공유 |
| **JSON** | API 통합 |

### 단일 노트 내보내기

1. 노트 열기
2. **파일** → **내보내기** 클릭
3. 형식 선택
4. 위치 선택
5. 저장

### 여러 노트 내보내기

1. 사이드바에서 노트 선택 (Cmd/Ctrl+클릭)
2. 마우스 오른쪽 버튼 → **선택 항목 내보내기**
3. 형식 선택
4. 위치 선택
5. 저장

### 모든 노트 내보내기

1. **파일** → **모두 내보내기** 열기
2. 형식 선택
3. 위치 선택
4. 내보내기 완료 대기

## 마이그레이션 가이드

### Obsidian에서

Classic은 Obsidian 볼트를 직접 가져올 수 있습니다

1. **파일** → **가져오기** 열기
2. **Obsidian 볼트** 선택
3. 볼트 폴더로 이동
4. 가져올 항목 선택

**전송되는 것**

- ✅ 노트 및 폴더
- ✅ 태그
- ✅ 링크 (백링크로 변환)
- ✅ 메타데이터
- ⚠️ 플러그인 (지원되지 않음)

### Notion에서

1. Notion에서 Markdown으로 내보내기
2. Classic으로 가져오기
3. 서식 검토
4. 끊어진 링크 수정

**Notion 마이그레이션 팁**

- Notion의 "Markdown으로 내보내기" 사용
- 체크박스 서식 확인
- 데이터베이스 참조 확인
- 연결된 데이터베이스 수동으로 다시 만들기

### Evernote에서

1. Evernote에서 HTML로 내보내기
2. Classic의 HTML 가져오기 사용
3. 변환된 콘텐츠 검토

**Evernote 마이그레이션 팁**

- 노트북이 폴더가 됨
- 태그가 보존됨
- 첨부 파일이 가져와짐
- 서식 확인

## 모범 사례

### 가져오기 전

- 기존 노트 백업
- 가져오기 미리보기 검토
- 중복 확인
- 서식 확인

### 가져오기 후

- 가져온 노트 검토
- 서식 문제 수정
- 내부 링크 업데이트
- 필요에 따라 태그 추가

### 정기 내보내기

- 매월 백업 내보내기
- 다른 용도에 다른 형식 사용
- 날짜별로 내보내기 정리
- 오프사이트 백업을 위해 클라우드에 저장

## 문제 해결

### 가져오기 실패

**일반적인 원인**

- 지원되지 않는 형식
- 손상된 파일
- 권한 문제
- 메모리 제한

**해결 방법**

1. 다른 형식 시도
2. 파일 무결성 확인
3. 읽기 권한 확인
4. 한 번에 더 적은 파일 가져오기

### 서식 손실

**일반적인 원인**

- 복잡한 서식
- 지원되지 않는 구문
- 변환 제한

**해결 방법**

1. 소스 앱에서 Markdown 내보내기 사용
2. 가져오기 전 서식 단순화
3. 가져오기 후 수동으로 수정

### 첨부 파일 누락

**일반적인 원인**

- 가져오기에 첨부 파일이 포함되지 않음
- 잘못된 첨부 파일 경로
- 크기 제한

**해결 방법**

1. 첨부 파일과 함께 다시 가져오기
2. 첨부 파일 폴더 확인
3. 첨부 파일을 별도로 가져오기
