# 로컬-퍼스트 아키텍처

Classic은 개인정보를 기반으로 구축되었습니다. 데이터는 항상 기기에 남습니다.

## "로컬-퍼스트"란 무엇인가요?

로컬-퍼스트는 다음을 의미합니다:

1. **데이터는 로컬에 저장됩니다** 컴퓨터의 폴더에 모든 노트, 첨부 파일 및 설정이 저장됩니다.
2. **클라우드가 필요하지 않습니다** Classic은 인터넷 연결 없이 작동합니다.
3. **데이터를 소유합니다** 데이터에 대한 완전한 제어권을 가집니다.

## 데이터 저장소

### 기본 위치

데이터는 전용 폴더에 저장됩니다:

| 플랫폼 | 위치 |
| ------ | ---- |
| **macOS** | `~/Library/Application Support/Classic/notes` |
| **Windows** | `%APPDATA%\Classic\notes` |
| **Linux** | `~/.config/Classic/notes` |

### 데이터 형식

- 노트는 일반 텍스트 파일 (`.md`)로 저장됩니다
- 첨부 파일은 `attachments` 하위 폴더에 저장됩니다
- 설정은 `settings.json`에 저장됩니다

### 파일 구조

```
notes/
├── 수신함/
├── 데일리 노트/
│   └── 2024-01-15.md
├── 프로젝트/
│   ├── 내 프로젝트.md
│   └── 서브-노트.md
└── attachments/
    └── image.png
settings.json
```

## 개인정보 장점

### 계정 불필요

- 가입 불필요
- 이메일 인증 불필요
- 개인정보 수집 없음

### 추적 없음

- 분석 없음
- 원격 측정 없음
- 외부 서버로 전송되는 사용 데이터 없음

### 오프라인 가능

- 인터넷 없이 작동
- 선택할 때만 동기화
- 데이터에 항상 액세스 가능

## 데이터 보안

### 사용자의 책임

- **백업**: 설정에서 자동 백업 설정
- **내보내기**: 정기적으로 데이터 내보내기
- **업데이트**: 보안 수정을 위해 Classic 업데이트 유지

### 당사의 책임

- **로컬 저장소**: 데이터를 보거나 건드리지 않음
- **백도어 없음**: 숨겨진 데이터 수집 없음
- **투명성**: 데이터 처리에 대해 공개적이고 정직함

## 동기화 (선택 사항)

Classic에는 기본 제공 클라우드 동기화가 없습니다. 그러나 데이터를 동기화할 수 있습니다:

### iCloud/OneDrive/Dropbox 사용

1. Classic 설정 열기
2. 노트 폴더 위치를 클라우드 폴더로 변경
3. Classic이 클라우드 제공업체를 통해 동기화됩니다

### Git 사용

1. 노트 폴더에 git 저장소 초기화
2. 수동으로 또는 자동화로 변경 사항 커밋 및 푸시

## 데이터 복구

### 백업에서

1. 설정 열기
2. 백업 및 복원으로 이동
3. 백업 파일 선택
4. 복원 클릭

### 수동 복구

1. 노트 폴더 찾기
2. 안전한 위치로 파일 복사
3. 새 Classic 설치로 가져오기

## 자주 묻는 질문

### 내 데이터가 정말 비공개인가요?

**네.** 타사 서비스를 사용하여 명시적으로 동기화하지 않는 한 데이터가 기기를 떠나지 않습니다.

### 데이터를 다른 컴퓨터로 이동할 수 있나요?

**네.** 노트 폴더를 새 컴퓨터로 복사하고 설정에서 Classic이 가리키도록 하면 됩니다.

### Classic을 제거하면 어떻게 되나요?

**데이터는 유지됩니다.** Classic을 제거해도 노트 폴더가 삭제되지 않습니다. 데이터 손실 없이 안전하게 제거 및 재설치할 수 있습니다.

### Classic이 내 데이터에 액세스할 수 있나요?

**아니요.** Classic은 데이터 수집을 위한 외부 연결이 없는 로컬 애플리케이션입니다.
