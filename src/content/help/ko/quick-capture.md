# 빠른 캡처

워크플로를 중단하지 않고 아이디어를 즉시 캡처하세요. 빠른 캡처는 시스템의 어디서나 최소한의 노트 작성 창을 엽니다.

## 전역 단축키

지정된 단축키를 눌러 빠른 캡처 열기

| 플랫폼 | 기본 단축키 |
| ------ | ----------- |
| **macOS** | `Cmd+Shift+P` |
| **Windows** | `Ctrl+Shift+P` |
| **Linux** | `Ctrl+Shift+P` |

설정에서 이 단축키를 사용자 정의할 수 있습니다.

## 빠른 캡처 사용법

1. 전역 단축키 누르기 (시스템의 어디서나)
2. 팝업 창에 노트 입력
3. `Enter`를 눌러 저장
4. `Escape`를 눌러 취소

## 기능

### 미니멀 인터페이스

- 작고 집중된 창
- Markdown 지원
- 입력에 자동 포커스
- 키보드 우선 설계

### 빠른 작업

- `# ` + 공백 = 제목
- `#수신함` = 수신함에 추가
- `#오늘` = 오늘의 노트에 추가
- `#태그이름`으로 태그

### 기본 대상

빠른 캡처 노트는 기본적으로 **수신함**으로 이동합니다. 이를 변경할 수 있습니다:

1. 설정 열기
2. 빠른 캡처로 이동
3. 기본 대상 선택

## 팁

### 캡처 워크플로

1. **캡처** - 아이디어를 빠르게 적기
2. **처리** - 매일 수신함 검토
3. **정리** - 적절한 폴더/노트로 이동

### 모범 사례

- 간단하게 유지 - 캡처당 하나의 아이디어
- 컨텍스트를 위해 태그 사용
- 서식 걱정하지 말기 - 나중에 다듬기
- 캡처를 정기적으로 검토 및 처리

### 통합

빠른 캡처는 다음과 잘 작동합니다:

- **데일리 노트**: 매일 캡처를 검토하며 시작
- **수신함**: 대기 영역으로 사용
- **태그**: 즉시 컨텍스트 추가

## 문제 해결

### 단축키가 작동하지 않나요?

**macOS**
1. 시스템 환경설정 > 키보드 > 단축키 확인
2. 충돌 찾기
3. Classic 재시작

**Windows**
1. 관리자 권한으로 Classic 실행
2. 충돌하는 단축키 확인
3. Classic 재시작

### 창이 나타나지 않나요?

- Classic이 실행 중인지 확인
- 단축키 확인
- 포커스 문제 확인
- Classic 재시작

### 캡처가 저장되지 않나요?

- 기본 대상 확인
- 쓰기 권한 확인
- 디스크 공간 확인
- Classic 재시작
