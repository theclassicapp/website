# الملاحظات الفرعية
تتيح إنشاء ملاحظات متداخلة هرميةية في كلاسيك. نظم المحتوى المعقد إلى أجزاء قابلة للإدارة.

## ما هي الملاحظات الفرعية؟
الملاحظات الفرعية هي ملاحظات داخل ملاحظات - طريقة لإنشاء هياكل محتوى هرمي.
### الفوائد
- **التنظيم**: تقسيم المواضيع المعقدة إلى أجزاء أصغر
- **التداخل**: إنشاء مستويات متعددة من التسلسل الهرميي
- **التنقل**: سهولة التنقل بين الملاحظات ذات الصلة
- **المرونة**: توسيع وطي حسب الحاجة
## إنشاء الملاحظات الفرعية
### الطريقة 1: المسافة البادئة
إنشاء ملاحظات فرعية بزيادة محتوى مساف بادئة للملاحظة الأصلية
1. أنشئ ملاحظة أصلية
2. أنشئ ملاحظة جديدة
3. اسحب الملاحظة الجديدي إلى الأصلية
4. تصبح ملاحظة فرعية
### الطريقة 2: قائمة السياق
1. انقر بزر الفأرة الأيمن على الملاحظة
2. حدد "New Subnote"
3. يتم إنشاء ملاحظة متداخلة
### الطريقة 3: اختصار لوحة المفاتيح
1. حدد ملاحظة
2. اضغط `Cmd/Ctrl+Shift+N`
3. يتم إنشاء ملاحظة فرعية
## إدارة الملاحظات الفرعية
### التوسيع والطي
انقر على السهم بجانب ملاحظة ذات ملاحظات فرعية
- **توسيع**: إظهار جميع الملاحظات المتداخلة
- **طي**: إخفاء الملاحظات المتداخلة
### التنقل
- انقر لفتح ملاحظة فرعية
- استخدم مفاتيح الأسهم للتنقل في التسلسل الهرمي
- يظهر مسار التنقل (breadcrumbs)
### نقل الملاحظات الفرعية
اسحب وأفلت لإعادة التنظيم
1. انقر واستمر الضغط على الملاحظة الفرعية
2. اسحب إلى موقع جديد
3. أفلت لإعادة التموضيع
### التحويل إلى ملاحظة رئيسية
ترقية ملاحظة فرعية إلى المستوى الأعلى
1. انقر بزر الفأرة الأيمن على الملاحظة الفرعية
2. حدد "Move to Root"
3. تصبح الملاحظة ملاحظة رئيسية
## عمق الملاحظات الفرعية
يدعم كلاسيك مستويات متعددة من التداخل
```
📁 Project
  📄 Overview
  📁 Research
    📄 User Interviews
    📄 Survey Results
    📁 Analysis
      📄 Key Findings
      📄 Recommendations
  📁 Planning
    📄 Timeline
    📄 Resources
```
### أفضل الممارسات
- **الحد من العمق**: 3-4 مستويات كحد أقصى للوضوح
- **التجميع المنطقي**: جمّع المحتوى ذي الصلة معاً
- **التسمية الواضح**: استخدم أسماء وصفية للملاحظات الفرعية

## الربط مع الملاحظات الفرعية
### الروابط الخلفية
تنشئ الملاحظات الفرعية تلقائياً روابط خلفية للملاحظات الأصلية
- شاهد قسم "Linked References"
- تنقل بين الملاحظات ذات الصلة
- فهم علاقات الملاحظات
### روابط ويكي
اربط للملاحظات الفرعية من أي مكان
```markdown
See [[Project/Research/Analysis]] for details.
```
## نصائح
### اصطلاح التسمية
استخدم تسمية واضح ومتسق
```
✅ Good:
Project/Requirements/Functional
Project/Requirements/Non-Functional

❌ Avoid:
Project/req/func
Project/rq/nf
```
### التنقل عبر مسار التنقل (breadcrumbs)
استخدم مسار التنقل لفهم موقعك
```
Project > Research > Analysis
```
### البحث
يشمل البحث الملاحظات الفرعية تلقائياً
- البحث في المحتوى عبر جميع المستويات
- التصفية حسب عمق الملاحظة
- تنقل سريع
### القوالب
تطبيق القوالب على الملاحظات الفرعية
1. أنشئ ملاحظة فرعية
2. استخدم أمر الشرطة المائلة `/template`
3. اختر القالب
## اختصارات لوحة المفاتيح
| الإجراء | الاختصار |
| -------- | ---------- |
| ملاحظة فرعية جديدة | `Cmd/Ctrl+Shift+N` |
| توسيع الكل | `Cmd/Ctrl+Down` |
| طي الكل | `Cmd/Ctrl+Up` |
| تحريك لأعلى | `Opt/Alt+Up` |
| تحريك الأسفل | `Opt/Alt+Down` |
| ترقية | `Opt/Alt+Left` |
| تخفي | `Opt/Alt+Right` |
## استكشاف الأخطاء
### الملاحظة الفرعية لا تظهر
- تحقق مما إذا كان الأصل مطوي
- تحقق مما إذا تم نقل الملاحظة
- حدّث الشريط الجانبي
### لا يمكن إنشاء ملاحظة فرعية
- تأكد من وجود الملاحظة الأصلية
- تحقق من الأذونات
- جرب طريقة قائمة السياق
### الملاحظات الفرعية المفقودة
استخدم البحث للعثور على الملاحظات المفقودة
1. افتح البحث (`Cmd/Ctrl+F`)
2. اكتب اسم الملاحظة
3. شاهد المسار الكامل في النتائج
